# TASKS — 작업 원장 (Task Ledger · 시간 기록)

> **목적**: 해야 할 일 / 한 일 / 추가된 일 / 변경·버그를 **한 표에서 시간 기록과 함께** 추적한다.
> **관계**: 범위·우선순위 백로그 = [TODO.md](TODO.md) · 버그 상세 = [BUGS.md](BUGS.md) · 진행 서술 = [journal/](journal/). 이 문서는 **각 Task의 생애주기 일시**를 단일 원장으로 모은다.
> **일시 규약**: `YYYY-MM-DD HH:MM`(KST). 개발/완료 일시는 **git 커밋 시각**, 설계는 설계 커밋, 테스트는 **CI 빌드 green** 또는 **실기 QA** 시각. 미정은 `—`.
> **유형**: 🐞버그 · ✨기능 · 📐설계 · ♻️리팩터.  **상태**: 요청 → 설계 → 개발 → CI빌드 → 실기QA → ✅완료 / ⏸보류.
> **주의**: WinUI는 맥 빌드 불가 → "테스트"는 **CI(빌드) green**까지가 자동 검증, 런타임 동작은 **실기 QA(Windows)** 로 별도 확인. 미확인은 "QA 대기".

## 5. 갱신 규약

- 새 Task는 **추가 일시**와 함께 이 표에 등록(상태 ☐/📐).
- 상태 전이 때 해당 일시 칸을 채운다: **설계**(설계 커밋) · **개발**(dev 커밋) · **테스트**(CI green/실기 QA) · **완료**(실기 QA 확인 또는 머지).
- 버그는 §2에 리포트 일시로 등록, 수정 커밋·완료 일시 기록. 상세 원인은 [BUGS.md](BUGS.md).
