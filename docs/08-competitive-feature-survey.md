# 08 · Mac/Windows 파일 탐색기 기능 조사 & 채택 추천

> 주요 파일 관리자들의 기능을 ① **각 프로그램의 강점(시그니처)** 과 ② **보편적으로 구현된 기능(테이블 스테이크)**
> 으로 정리하고, Nexa Dir이 **채택할 항목을 우선순위로 추천**한다.
> (지식 기준 정리 — 버전별 차이 가능, 검증되면 갱신.)

## 2. 보편적으로 구현된 기능 (테이블 스테이크 — 필수 기본기)

> 어느 제품이든 기대되는 기능. **없으면 감점**, 있다고 차별화는 안 됨.

- 다중 보기 모드: 목록 / 디테일 / 아이콘 / 타일·그리드
- 좌측 **내비게이션 트리**(사이드바) + 즐겨찾기/북마크
- **탭** 브라우징
- 정렬 / 그룹 / 필터
- 복사·이동·삭제·이름변경, **드래그앤드롭**, 잘라내기/붙여넣기
- **휴지통**, 실행취소
- **검색**(이름/내용), 주소줄 / **브레드크럼**(경로 바)
- **썸네일·미리보기**, 속성 보기
- 컨텍스트(우클릭) 메뉴, 새 폴더
- 숨김 파일 토글, 키보드 내비게이션
- 기본 네트워크/클라우드 접근

→ Nexa Dir 매핑: **M0/M1**에서 대부분 충족(FR-A~D, FR-J). 표 스테이크는 "빠르고 견고하게"가 관건.

## 3. 프로그램별 강점(시그니처) 기능

| 기능 | 대표 제품 | 한 줄 설명 | 평가받는 이유 |
| --- | --- | --- | --- |
| **목록 인라인 펼침**(disclosure) | Finder | 목록 보기에서 폴더를 그 자리에서 트리로 펼침 | 진입 없이 하위 탐색 |
| **컬럼(Miller) 뷰** | Finder, QSpace, One Commander | 계층을 좌→우 다단 컬럼으로 | 깊은 경로 빠른 탐색·미리보기 |
| **Quick Look**(스페이스바) | Finder | 선택 즉시 전체 미리보기 오버레이 | 앱 안 열고 즉시 확인 |
| **Drop Stack / 스테이징 바구니** | Path Finder | 여러 곳의 파일을 바구니에 모아 일괄 작업 | 흩어진 파일 모으기 |
| **Branch / Flat View** | XYplorer, Opus | 하위 폴더 전체를 한 목록으로 평탄화 | 대량 파일을 폴더 무관하게 한눈에 |
| **원격/클라우드 마운트·동기화** | ForkLift, TC | SFTP/S3/WebDAV를 로컬처럼 + 폴더 동기화 | 전송 워크플로우 일류 |
| **즉시 파일명 검색** | Everything | USN 저널로 전 디스크 즉시 검색 | 체감 0ms 검색 |
| **태그/색상 라벨** | Finder, XYplorer | 색·태그로 분류·필터 | 폴더 구조 밖 분류 |
| **고급 일괄 이름변경** | Opus, TC, ForkLift | 패턴/정규식/메타 기반 + 미리보기 | 대량 정리 |
| **폴더 비교·동기화** | TC, Opus | 좌우 디렉터리 diff·미러 | 백업·정리 |
| **스크립팅/플러그인** | Opus, TC, XYplorer | 사용자 자동화·확장 | 무한 확장성 |
| **내장 터미널** | Path Finder, Commander One | 현재 폴더에서 셸 | 개발자 워크플로우 |
| **듀얼 패널 + 키보드 중심** | TC, Opus, Marta | F5/F6 전송, 키보드 완결 | 고속 파일 작업 |
| **스마트 폴더/저장된 검색** | Finder | 조건을 폴더처럼 저장 | 동적 뷰 |
| **메타데이터 패널** | Opus, One Commander | EXIF/속성/해시 사이드 | 미디어·문서 정리 |
| **모던 Fluent UI** | Files, One Commander | 깔끔한 디자인·다크 | 기본 탐색기 대비 인상 |

## 4. Nexa Dir 채택 추천 (우선순위)

> 전략: **테이블 스테이크는 빠르고 견고하게**, **시그니처는 선별 채택 + 우리만의 결합**으로 차별화.

### 4-1. 이미 우리 설계의 핵심 (확정)
- ✅ **목록 인라인 펼침 × 폴더 교차 다중 선택 × 혼합 일괄 작업** — Finder의 펼침을 넘어
  *서로 다른 폴더 동시 선택*까지 = **시그니처 0** (M1, `07`).
- ✅ 듀얼 패널 + 패널별 탭(F5/F6, 키보드) — TC/Path Finder 강점 (M1).
- ✅ Quick Look 미리보기 (M2), 즉시 검색(Everything식, M3), 원격/클라우드(ForkLift식, M4),
  내장 터미널/런처(Path Finder식, M1 런처/P2 패널), 컨텍스트 메뉴(M1).

### 4-2. 강력 추천 — 로드맵에 추가 권장 (P1)
| # | 기능 | 출처 | 채택 이유 / Nexa 결합 |
| --- | --- | --- | --- |
| R1 | **컬럼(Miller) 뷰** | Finder/QSpace | 인라인 트리와 별개의 깊은 경로 뷰. 보기 모드로 추가 |
| R2 | **Branch/Flat View** | XYplorer/Opus | "하위 전체 평탄화" — 우리 *가시 노드 스트림* 모델과 자연 결합(펼침의 극단형) |
| R3 | **Drop Stack(선택 바구니)** | Path Finder | 교차폴더 다중선택과 시너지 — 흩어진 선택을 모아 일괄 작업 |
| R4 | **태그/색상 라벨** | Finder/XYplorer | 검색 필터·정리와 결합, 폴더 밖 분류 |
| R5 | **고급 일괄 이름변경** | Opus/TC | 패턴/정규식/메타 + 실시간 미리보기 |
| R6 | **폴더 비교·동기화** | TC/Opus | 듀얼 패널과 결합한 diff/미러 |

### 4-3. 후속 추천 (P2~)
- R7 **스마트 폴더/저장된 검색**(Finder) — 검색 결과를 동적 폴더로.
- R8 **메타데이터 패널**(Opus) — 미리보기와 결합.
- R9 **스크립팅/플러그인 API**(Opus/TC) — 확장성 장기 과제.
- R10 **아카이브 인플레이스 탐색**(TC) — zip/7z를 폴더처럼.

### 4-4. 의도적 비채택/보류
- 4분할 등 과도한 다중 패널(Q-Dir) → 듀얼 + 패널별 탭으로 충분, 복잡도 회피.
- 무거운 리본/모듈 과잉(구형 Opus) → 미니멀 + 명령 팔레트로 대체.

## 5. 차별화 한 줄 요약

> 기존 제품은 **성능(TC/Opus)** 또는 **모던 UX(Files/Finder)** 중 하나에 치우침.
> Nexa Dir = **네이티브 성능** × **모던 Fluent** × **(인라인 펼침 + 폴더 교차 다중 선택)** 의 결합 —
> 어느 단일 제품도 매끄럽게 제공하지 못하는 조합이 시그니처.
