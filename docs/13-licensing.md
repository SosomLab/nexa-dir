# 13 · 라이선스 방향 검토 (OD3) — 차후 유료 판매 가능

> 목표: **현재 설계(안 A: Rust 코어 + WinUI 3, 프로툴 고밀도, MSIX 배포)를 고려해
> 차후 유료 판매가 가능한 라이선스 방향을 미리 결정**.
> 상태: **확정(2026-06-30)** — 아래 §0.

## 0. 확정 (DR-5)

- **방향**: **소스공개 제한형(Source-available)** — 소스는 GitHub 공개, **상업적 사용만 유료**.
- **권장 라이선스**: **PolyForm Noncommercial 1.0.0** (개인·비상업 무료 / 상업 사용 유료 라이선스 판매).
  - 대안: **BSL 1.1**(비프로덕션 무료/프로덕션 유료 + Change Date 후 Apache-2.0 자동 전환) — 커뮤니티 신뢰 중시 시.
- **무료 버전**: 개인·비상업 **무료**, 상업 사용 **유료**(라이선스에 내재) → 별도 무료/유료 경계 설계 부담 최소.
- **사용자 답변 근거**: 소스=공개, 무료버전=추천 위임 → 공개+유료 보호에 가장 적합한 소스공개 제한형 채택.
- **즉시 시행(§4)**: 소유권/CLA·상표·퍼미시브 의존성 정책·CI 라이선스 게이트·NOTICE.

---

## 1. "유료 판매 가능"의 두 전제

1. **우리 코드 권리 보유** — 자유롭게 팔 수 있으려면 *우리 코드*에 **카피레프트(GPL/AGPL)를 쓰지 않고**,
   저작권/재라이선스 권리를 우리가 보유해야 함.
2. **의존성 라이선스 호환** — 번들/링크하는 OSS가 **상용·비공개 배포를 허용**해야 함(퍼미시브 위주).
   → 이 점이 안 지켜지면 *모델과 무관하게* 유료 판매가 막힐 수 있음 → **지금부터 통제**.

## 2. 스택 의존성 라이선스 점검 (현 설계 기준)

| 구성 | 라이선스 | 상용/유료 판매 |
| --- | --- | --- |
| WinUI 3 / Windows App SDK | **MIT** | ✅ 재배포 가능 |
| .NET 8 / 런타임 | **MIT** | ✅ |
| Rust std/툴체인 | **MIT/Apache-2.0** | ✅ |
| tantivy(검색) | **MIT** | ✅ |
| notify(감시) | **MIT/CC0** | ✅ |
| image(이미지) | **MIT/Apache-2.0** | ✅ |
| zip(압축) | **MIT** | ✅ |
| ssh2/libssh2(SFTP) | **BSD-3** | ✅ |
| aws-sdk(S3) | **Apache-2.0** | ✅ |
| wasmtime(플러그인) | **Apache-2.0 (LLVM 예외)** | ✅ |
| mlua(Lua) | **MIT** | ✅ |
| Python/Node(플러그인 런타임, 번들 시) | **PSF / MIT** | ✅ |

> **결론:** 현 설계 의존성은 **모두 퍼미시브** → 비공개/유료 판매 호환. 단 **잠재 위험** 관리 필요:
> - **GPL/AGPL 크레이트 금지**(우리 바이너리에 카피레프트 전이).
> - **LGPL(예: FFmpeg 미디어 디코딩)** → 동적 링크 또는 **아웃오브프로세스 격리**로만 사용.
> - **unRAR 독점 조항**(rar) → 내장 금지, 필요 시 플러그인/외부 도구. **1차 압축은 zip(MIT)만 내장**.

## 3. 라이선스 모델 후보 (유료 판매 관점)

| 모델 | 설명 | 유료화 | 예시 | 비고 |
| --- | --- | --- | --- | --- |
| **A. 독점(Closed)** | 비공개, All Rights Reserved | 직접 판매 | Directory Opus, XYplorer, Path Finder | 통제 최대, 커뮤니티/기여 없음 |
| **B. 오픈코어** | 퍼미시브(MIT/Apache) 코어 + 유료 프리미엄(비공개) | 프리미엄 판매 | 다수 SaaS/툴 | 커뮤니티+수익, 모듈 경계 필요 |
| **C. 소스공개(Source-available)** | 소스 공개·상용/경쟁 사용 제한, 라이선스 판매 | 상용 라이선스 | BSL 1.1, PolyForm | 투명성+수익, 기한 후 오픈 전환 가능(BSL) |
| **D. 듀얼 라이선스** | GPL + 상용 예외 판매 | 상용 예외 | Qt, MySQL | CLA 필요, 복잡 |

## 4. 권장

> **1차 권장: A. 독점(Closed-source) — 비공개 상용.**
> 경쟁 제품(Opus/XYplorer/Path Finder)이 모두 독점 유료이고, 목표가 "유료 판매"이므로 가장 단순·직접적.
> 커뮤니티/투명성을 원하면 **C. 소스공개(BSL/PolyForm)** 가 차선 — 소스 공개하되 상용 판매권 유지.
> **B. 오픈코어**는 무료 사용자 확보 후 프리미엄 판매를 원할 때.

### 모델과 무관하게 "지금" 확정·시행할 것 (유료화 보호 장치)
1. **저작권/소유권 보유** — 외부 기여 받으면 **CLA/DCO**로 재라이선스·판매 권리 확보.
2. **상표(브랜드) 보호** — 제품명/로고는 코드 라이선스와 **별도 상표**로 보호(소스공개여도 브랜드 통제).
3. **의존성 정책: 퍼미시브 온리** — 허용(MIT/Apache/BSD/ISC/MPL-2.0) 화이트리스트,
   **GPL/AGPL 거부**, LGPL 검토. **CI 라이선스 게이트**(`cargo-deny`, .NET 라이선스 스캔)로 강제.
4. **서드파티 고지(NOTICE)** — 퍼미시브 의존성 저작권 고지 파일 유지.
5. **유료 바이너리 분리 여지** — 어떤 모델이든 *배포 바이너리/프리미엄*은 비공개 가능하게 모듈 경계 유지.

## 5. 결정 시 영향

- **소스 공개 범위**: A=비공개, B=코어만, C=전체(제한부), D=GPL.
- **저장소 운영**: A는 private 권장; B/C/D는 public + 라이선스 파일(LICENSE) + CLA.
- **배포/가격**: 모두 MSIX 유료 판매 가능(Store 유료/사이트 직판/라이선스 키).
