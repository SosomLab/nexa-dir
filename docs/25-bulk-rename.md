# 25 · 일괄 이름변경 (Batch Rename) 설계

> 목표: **다중 선택(교차폴더 포함) 파일/폴더를 규칙 파이프라인으로 일괄 리네임 + 실시간 미리보기**.
> Path Finder식 **빌딩 블록**(다단계 조합·저장/재사용) + **표현식/함수 언어**(문자열·메타데이터·널 처리) +
> **개발자 확장(UDF)**. 요구: FR-D5([05](05-requirements.md)) · 로드맵 M1 파일작업([02](02-roadmap.md)) · [TODO REN-1](TODO.md).
> 상태: **설계**(2026-07-08 사용자 스펙 반영). 구현은 단계적(§8).

---

## 1. 접근 & 실행

| 항목 | 내용 |
|---|---|
| 진입 | 파일/폴더 **다중 선택**(Shift/Ctrl/러버밴드) → **명령(C) 메뉴 · 컨텍스트 메뉴 · 명령 팔레트** → "일괄 이름 변경" · 단축키(예: F2 확장/`Ctrl+Shift+R`) |
| 대상 | 여러 파일·폴더 동시(교차폴더 — 플래그십 [07](07-flagship-tree-multiselect.md) 시너지) |
| 미리보기 | 대화상자 우측에 **원본 → 새 이름** 실시간 표(충돌 하이라이트) |

## 2. 이름 변경 동작 (Operations) — 6종

| # | 동작 | 기능 |
|---|---|---|
| 1 | **Replace Text** | 특정 문자열 → 다른 문자열 치환(대소문자 옵션·전체/첫 일치·이름/확장자 대상) |
| 2 | **Replace RegEx** | 정규식 패턴 치환(캡처 그룹 `$1`·`${name}`) |
| 3 | **Insert Text** | 지정 위치(접두/접미/인덱스/구분자 기준)에 텍스트 삽입 |
| 4 | **Change Case** | 대문자 / 소문자 / 제목식(Title) / 첫 글자(Sentence) |
| 5 | **Add Number Sequence** | 연번(시작값·증가폭·자릿수 0패딩·위치·정렬 기준) |
| 6 | **Add Date** | 파일 날짜(수정/생성/접근) 또는 **메타 날짜**(EXIF 촬영일 등, §5) — 형식 지정 |

- **확장자 처리**: 이름만 / 확장자 포함 / 확장자만 대상 선택(동작별).

## 3. 워크플로우 (조합·재사용)

| 항목 | 내용 |
|---|---|
| **빌딩 블록** | `+` 버튼으로 동작을 **다단계로 쌓아** 조합(예: 치환 → 연번 → 날짜) |
| **적용 순서** | 위→아래 **순차** 적용(각 단계의 출력이 다음 입력). 드래그로 순서 변경·on/off 토글 |
| **멀티파트 네이밍** | 여러 규칙 조합으로 복합 이름(`{촬영일}_{카메라}_{연번}.{ext}`) |
| **시퀀스 저장** | **Save Renaming Sequence** — 규칙 묶음(프리셋)을 이름 붙여 저장 → 재사용·공유(JSON, settings와 별도 `renames/*.json`) |

## 4. 표현식 · 함수 언어 (엔진 코어) ★

토큰 템플릿과 함수를 결합한 **작은 표현식 언어**를 `nexa-rename`에 내장한다(부수효과 없는 순수 변환).
"전체 교체(템플릿)" 동작과 각 동작의 인자에서 사용.

### 4-1. 토큰(변수)
`{name}` 원본 이름(확장자 제외) · `{ext}` 확장자 · `{basename}` 폴더/파일 전체 이름 · `{parent}` 상위 폴더명 ·
`{num}` 연번(동작 5) · `{date:FORMAT}` 파일 날짜 · **`{meta.KEY}`** 메타데이터(§5) · `{index}`/`{count}` 선택 순번/총수.

### 4-2. 문자열 함수 (요청 2-2)
- **추출**: `LEFT(s,n)` · `RIGHT(s,n)` · `SUBSTR(s,start,len)`(=MID) · `BETWEEN(s,a,b)` · `SPLIT(s,sep,i)`
- **검색/길이**: `LEN(s)` · `FIND(s,sub)` · `CONTAINS(s,sub)`
- **정리**: `TRIM(s)` · `REPLACE(s,a,b)` · `REMOVE(s,pat)` · `PAD(s,n,ch)`/`ZEROPAD(n,width)`
- **형식**: `UPPER`/`LOWER` · `PROPER`/`TITLE`
- **조합**: `CONCAT(...)` · `JOIN(sep,...)` · `IF(cond,a,b)`

### 4-3. 널/공백 처리 (요청 2-3)
- 함수: `IFNULL(x,alt)` · `COALESCE(a,b,…)` · `DEFAULT(x,d)` · `ISBLANK(x)` · `NULLIF(x,y)`
- **정책(전역 옵션)**: 빈 값 시 **대체 / 건너뛰기(skip) / 경고** 선택 · **연쇄 구분자 자동 정리**(`__`→`_`, 끝 구분자 제거) ·
  **전체 빈 이름 방지**(빈 결과는 원본 유지 + 미리보기 경고).

### 4-4. 평가 규약
- **결정성**: 순수 함수(입력=이름+메타+순번 → 출력 문자열). 시계·난수·I/O 없음(연번·날짜는 사전 계산해 주입).
- **오류 격리**: 항목별 평가 실패 → 그 항목만 "오류"로 미리보기 표시(전체 중단 안 함).
- **구현**: `nexa-rename`에 **손수 만든 소형 평가기**(토큰 파서 + 함수 테이블) — 스크립트 엔진 불요, 가장 빠르고 안전. 90% 커버.

## 5. 메타데이터 변수 시스템 (요청 2-1) ★ — 리네임 + 정보 패널 공용

파일 내부 임베디드 메타를 **`{meta.KEY}` 변수**로 노출. **동일 추출기가 하단 정보 패널(BP-2, [35](35-preview-system.md))의
"자체 Meta 포맷 파일 정보 표시"도 담당**(요청: 정보 표시 개선). → 코어 `nexa-meta` 크레이트로 단일화.

| 포맷군 | 대상 | Rust 크레이트(퍼미시브) | 노출 키(예) |
|---|---|---|---|
| 사진 | EXIF·IPTC·XMP(JPEG/TIFF/HEIC/RAW) | `kamadak-exif` · `nom-exif` · `quick-xml`(XMP) | `meta.DateTaken` `meta.Camera` `meta.ISO` `meta.GPS` |
| 동영상 | QuickTime/MP4 atom | `mp4` / `mp4parse` | `meta.Duration` `meta.Resolution` `meta.Codec` `meta.CreationDate` |
| 오디오 | ID3(MP3)·Vorbis(FLAC/OGG)·MP4 | **`lofty`**(다포맷 단일 API) | `meta.Artist` `meta.Title` `meta.Album` `meta.Track` |
| 문서 | PDF · OOXML(docx/xlsx/pptx) | `lopdf`/`pdf` · `zip`+`quick-xml`(docProps/core.xml) | `meta.Author` `meta.Title` `meta.Pages` `meta.Created` |

- **아키텍처**: `nexa-meta`가 확장자/매직바이트로 provider 선택 → **키-값 딕셔너리** 반환(맥 단위테스트 가능·부수효과 없음).
  리네임은 `{meta.KEY}`로, 정보 패널은 같은 딕셔너리를 속성 표로 렌더.
- **성능**: 지연/부분 파싱(헤더만) · 캐시(경로+mtime 키). 배치 리네임은 선택 항목만 1회 추출.
- **널 처리**: 없는 키 → 널(§4-3 정책 적용). 정보 패널은 있는 키만 표시.
- **의존성 정책**: 전부 **퍼미시브(MIT/Apache) 순수 Rust 우선**. C 의존(gexiv2/rexiv2 등)은 배포·크로스 부담이라 회피.

## 6. 개발자 확장 — 사용자 정의 함수(UDF) 스택 ★ (요청 2-4)

> 요청: "Python 커스텀 함수(플러그인 파일 방식)". 단, **CPython 임베드(PyO3)는 부적합** — 런타임 번들(배포·포터블
> 훼손)·**샌드박스 불가**(임의 import/FS/네트워크)·비결정성·C 의존. 아래는 **Rust 임베드 인터프리터** 기준 추천.

### 6-0. 계층(대부분은 스크립트 불요)
- **Tier 0 — 내장 DSL(§4)**: 토큰+함수. 스크립트 엔진 없이 Rust 네이티브. 결정적·최속·안전. **90% 커버**.
- **Tier 1 — UDF 스크립트(권장)**: 사용자가 `def rename(name, meta, ctx): ...` 선언·호출. 아래 스택.
- **Tier 2 — 풀 플러그인**: 무겁거나 상태 있는 확장은 M6 플러그인(WASM/Lua, [09](09-plugin-architecture.md)).

### 6-1. UDF 요구 & 후보 비교
요구: **순수 문자열/메타→문자열 · 샌드박스(무 I/O) · 결정적 · 배치(수천 파일) 저비용 · 오류 격리 · 순수 Rust(무 C 의존, 포터블)**.

| 엔진 | 문법 | 샌드박스 | 결정성 | 성능(배치) | 순수Rust | 평가 |
|---|---|---|---|---|---|---|
| **Rhai** | 자체(JS/Rust풍) | ★ 기본 무 I/O·노출 API만 | ★ `max_operations` fuel·무 ambient | ★ AST 1회 컴파일→재사용 | ★ | **범용 임베드 1순위** |
| **Starlark**(starlark-rust) | **Python 유사** | ★ 하메틱(무 I/O·frozen) | ★★ 설계상 결정적(Bazel/Buck) | ○ | ★ | **"Python 문법+결정성" 최적** |
| **mlua**(Lua) | Lua | ○ io/os 제거 시 | ○ 샌드박스 시 | ★ 초경량 | ★(vendored) | **M6와 엔진 공유(추가 의존 0)** |
| WASM(wasmtime) | 임의(컴파일) | ★★ 최강·fuel | ★★ | ○(호출 오버헤드) | ★ | 저작 마찰 큼 → M6 무거운 확장용 |
| CPython/PyO3 | Python | ✕ | ✕ | ○ | ✕(C·런타임) | **부적합**(배포·보안·결정성) |
| RustPython | Python(순수Rust) | ○ | ○ | △ 느림·미완 | ★ | 후속 "파이썬맛" 옵션 여지 |

### 6-2. 권장
1. **기본 권장 = Starlark(starlark-rust)** — 사용자의 "Python 커스텀 함수" 취지를 CPython 문제 없이 만족(파이썬 유사 문법 +
   **설계상 결정적·하메틱 샌드박스** + 순수 Rust). 리네임 UDF는 "이름+메타 → 이름"의 순수 함수라 Starlark 제약(무 I/O·유한 루프)이 **오히려 안전 보장**.
2. **범용 임베드가 더 중요하면 = Rhai** — fuel 제한·AST 재사용으로 배치 최속, 노출 API 세밀 제어.
3. **추가 의존 회피 = mlua(Lua)** — M6가 이미 Lua 채택([09](09-plugin-architecture.md)) → 엔진 공유. 문법 친숙도만 낮음.

> **결정 방향**: 리네임 UDF는 **Starlark**(파이썬 문법·결정성), M6 범용 플러그인은 기존 **WASM+Lua** 유지 — 별도 ADR로 확정(구현 착수 시).

### 6-3. UDF 실행 모델 (공통)
- **플러그인 파일 방식**: `%APPDATA%\NexaDir\rename-udf\*.star`(또는 `.rhai`) — 함수 선언 파일을 드롭 → 리네임 대화상자
  "함수" 목록에 노출. 핫리로드(파일 변경 감지).
- **호출 계약**: `rename(name: str, ext: str, meta: dict, index: int, count: int) -> str`. 반환 빈/오류 → §4-3 널 정책.
- **격리**: 항목별 호출을 try/catch(패닉 경계) — 한 항목 실패가 배치를 중단하지 않음. **fuel/스텝 상한**으로 무한 루프 차단.
- **결정성**: 시계·난수·환경 접근 미노출. 필요한 컨텍스트(날짜·순번)는 호스트가 인자로 주입.
- **호스트 API(화이트리스트만)**: 문자열 함수(§4-2)·`meta` 읽기 전용. FS/네트워크/프로세스 **미노출**.

## 7. 미리보기 · 적용 · 안전장치

- **미리보기 표**: 항목별 `원본 → 새 이름`, **충돌**(중복·빈 이름·금지문자·기존 파일 존재) 하이라이트, 적용 전 검증.
- **적용**: `nexa-ops` 리네임에 (경로, 새 이름) 목록 전달 — **충돌 정책**(자동 `(2)`/건너뜀/중단)·바이트 무관 진행률.
- **실행취소**: 일괄 리네임을 **하나의 트랜잭션**(B-13u [OperationHistory])으로 Ctrl+Z. 앱 계층 현 구현 → nexa-ops 이관.
- **교차폴더**: 플래그십([07](07-flagship-tree-multiselect.md))으로 여러 폴더 파일을 한 번에.

## 8. 우리 스택 매핑 & 단계적 구현

| 구성 | 위치 / 기술 |
|---|---|
| 규칙 파이프라인 + 표현식/함수(§4) | **`nexa-rename`**(Rust, 순수 — 맥 단위테스트) |
| 메타데이터 추출(§5) | **`nexa-meta`**(Rust) — 리네임 + 정보 패널 공용 |
| UDF(§6) | **Starlark(권장)** 임베드 in `nexa-rename` (별도 ADR) |
| 충돌·일괄 적용·Undo | `nexa-ops`(리네임·트랜잭션) |
| UI | 배치 리네임 대화상자(빌딩 블록 편집 + 미리보기 표) — Windows |

- **α (M1 P1)**: 동작 6종(§2) + 빌딩 블록/순서(§3) + 실시간 미리보기 + 충돌 검출 + 적용/Undo. 표현식은 토큰만.
- **β (M2)**: 함수 언어(§4-2/4-3) + 프리셋 저장/재사용 + 확장자 분리 옵션.
- **γ (M2~)**: 메타데이터 변수(§5)+정보 패널 개선 · 정규식 캡처 치환.
- **δ (후속)**: UDF(§6, Starlark) · AI 이름 제안(M5 연계).
