# 25 · 일괄 이름변경 (Path Finder식 배치 리네임) 설계

> 목표: **다중 선택(교차폴더 포함) 파일을 규칙 파이프라인으로 일괄 리네임 + 실시간 미리보기**.
> 사용자가 macOS **Path Finder의 배치 리네임**을 선호 → 그 UX를 채택. 요구: FR-D5([05](05-requirements.md)). 구현 대상 등록.

---

## 1. Path Finder식 = 규칙 스택(순서 적용) + 실시간 미리보기

여러 **규칙(rule)** 을 순서대로 쌓아 원본 이름에 차례로 적용하고, **미리보기 표**(원본 → 새 이름)를 실시간 갱신한다.

| 규칙 | 동작 |
| --- | --- |
| **찾기/바꾸기** | 텍스트 또는 `정규식`(대소문자 옵션, 전체/첫 일치) |
| **텍스트 추가** | 접두/접미/지정 위치 삽입 |
| **제거** | 문자 수/범위/패턴 제거 |
| **대소문자 변경** | 대문자 / 소문자 / 제목식(Title) / 첫 글자 |
| **일련번호** | 시작값·증가폭·자릿수(0패딩)·위치, 정렬 기준 |
| **날짜/EXIF 삽입** | 파일 날짜 또는 사진 EXIF 촬영일(형식 지정) |
| **전체 교체(템플릿)** | 토큰으로 새 이름 구성: `%name% %ext% %num% %date% %parent%` |

- **확장자 처리**: 이름만/확장자 포함/확장자만 대상 선택.
- **프리셋**: 규칙 묶음 저장·재사용.

## 2. 미리보기 · 적용 · 안전장치

- **미리보기 표**: 각 항목 `원본 → 새 이름`, **충돌(중복/빈 이름/금지문자) 하이라이트**, 적용 전 검증.
- **적용**: `nexa-ops` 리네임에 (경로, 새 이름) 목록 전달. **충돌 정책**(자동 접미 `(2)`/건너뜀/중단), 진행률.
- **실행취소(Undo)**: FR-D4 — 일괄 리네임을 **하나의 트랜잭션**으로 되돌림.
- **교차폴더 시너지**: 플래그십(서로 다른 폴더 파일 동시 선택, [07](07-flagship-tree-multiselect.md))으로 **여러 폴더 파일을 한 번에** 리네임.

## 3. 우리 스택 매핑

| 구성 | 위치 / 기술 |
| --- | --- |
| 규칙 파이프라인(변환) | **Rust 코어 `nexa-rename`** — 순수 함수(입력 이름+메타 → 새 이름). **맥에서 단위테스트 가능** |
| 충돌 검출·일괄 적용 | 코어 `nexa-ops`(리네임) + 트랜잭션/Undo |
| UI | 배치 리네임 대화상자(규칙 스택 편집 + 미리보기 표) — Windows |
| 미리보기 | 코어가 (원본→새이름) 목록 계산 → 인터롭으로 UI 표시 |

> **맥 개발 이점**: 규칙 엔진이 **부수효과 없는 순수 변환**이라 `nexa-rename`을 맥에서 풍부히 단위테스트(케이스/정규식/번호/충돌) 가능. UI만 Windows.

## 4. 단계적 구현 (M1 P1 → M2)

- **α (핵심, M1 P1)**: 찾기/바꾸기·텍스트 추가/제거·대소문자·일련번호 + **실시간 미리보기** + 충돌 검출 + 적용/Undo.
- **β (M2)**: 날짜/EXIF 삽입·템플릿(토큰) 교체·프리셋 저장·확장자 분리 옵션.
- **γ (후속)**: 정규식 캡처 그룹 치환, 플러그인 규칙(FR-L 커스텀 규칙), AI 이름 제안(M5 연계).
