# 00 · 비전 & 목표

## 1. 한 줄 정의

> Windows 기본 탐색기를 대체하는, **네이티브 성능 + 차세대 UX**의 파일 관리자.

## 2. 문제 정의 (Why)

Windows 11 기본 탐색기는 다음 한계를 가진다.

- 대용량 폴더(수만 개 파일) 진입 시 지연·멈춤.
- 탭은 추가됐으나 **듀얼 패널·트리 멀티뷰**가 없어 파일 이동/비교 작업이 번거로움.
- 미리보기 창이 느리고 포맷 지원이 제한적.
- 검색이 느리고(인덱싱 의존) 자연어/내용 기반 검색 불가.
- 클라우드·원격(SFTP/S3) 저장소를 1급 시민으로 다루지 못함.

## 3. 목표 (Goals)

| # | 목표 | 성공 지표 (북극성) |
| --- | --- | --- |
| G1 | 즉시 반응하는 탐색 | 100k 항목 폴더 진입 후 **첫 렌더 < 150ms**, 스크롤 60fps |
| G2 | 강력한 파일 작업 UX | 탭·듀얼 패널·키보드 중심 워크플로우 |
| G3 | 빠른 미리보기 | 일반 이미지/문서 프리뷰 **< 100ms** |
| G4 | 빠른 검색 | 자체 인덱스로 파일명 검색 **< 50ms**, 내용·자연어 검색 지원 |
| G5 | 어디서나 동일 경험 | 로컬/클라우드/원격을 동일 UI로 마운트·탐색 |

## 4. 비목표 (Non-Goals, 초기)

- 파일 편집기(텍스트/이미지) 내장 — 외부 앱 연동으로 대체.
- macOS/Linux 정식 지원 — 코어는 이식 가능하게 설계하되 1차 타깃은 **Windows**.
- 시스템 셸 완전 대체(데스크톱/작업표시줄) — 탐색기 앱 범위로 한정.

## 5. 타깃 사용자

- **파워 유저/개발자**: 키보드 중심, 대용량/다계층 디렉터리, 원격 서버 작업.
- **크리에이터**: 대량 미디어 미리보기·정리.
- **일반 사용자**: 기본 탐색기보다 빠르고 직관적인 대체재.

## 6. 경쟁 제품 분석

| 제품 | 강점 | 약점 / 우리의 차별점 |
| --- | --- | --- |
| Windows File Explorer | 기본 내장, 셸 통합 | 느림, 듀얼 패널 없음, 확장성 낮음 |
| Files (files.community) | WinUI, 모던 UI, 탭 | 성능 한계(.NET/UWP 기반), 대용량 약함 |
| Directory Opus | 기능 방대, 듀얼 패널 | 유료, UI 구식, 무거움 |
| Total Commander | 강력한 듀얼 패널·플러그인 | UI 노후, 학습곡선 |
| One Commander | 모던 듀얼 패널 | 성능·확장성 제한 |

**차별화 포인트**: *네이티브 코어 성능* × *모던 Fluent UI* × *AI/클라우드 1급 통합*.
기존 제품은 "성능(TC/Opus)" 또는 "모던 UX(Files)" 중 하나에 치우쳐 있음 — 둘 다를 노린다.
