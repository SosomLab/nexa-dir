# 01 · 아키텍처 & 기술 스택

## 1. 기술 스택 결정 (ADR-0001)

### 요구사항

- **네이티브 + 최대 성능** (사용자 명시 요구).
- 모던 Fluent UI (탭/듀얼 패널/미리보기).
- AI(임베딩·추론), 클라우드/원격 통합 같은 무거운 백그라운드 작업.
- macOS에서 개발하되 타깃은 Windows.

### 후보 비교

| 스택 | 성능 | UI 생산성 | Windows 네이티브 통합 | AI/시스템 라이브러리 | 평가 |
| --- | --- | --- | --- | --- | --- |
| **C++/WinRT + WinUI 3** | ★★★★★ | ★★ | ★★★★★ | ★★★ | 최고 성능, 개발비 매우 큼 |
| **.NET 8/9 + WinUI 3 (C#)** | ★★★★ | ★★★★★ | ★★★★★ | ★★★★ | 모던 네이티브 UI, 생산성 높음 |
| **Rust 코어 + WinUI 3 UI** | ★★★★★ | ★★★★ | ★★★★★ | ★★★★★ | 핫패스 네이티브, UI 생산성 양립 |
| Electron / Tauri(웹UI) | ★★★ | ★★★★★ | ★★ | ★★★★ | 셸 통합·성능에서 네이티브 요구 미달 |

### 결정 — 하이브리드 네이티브

> **코어 엔진: Rust** + **UI 셸: WinUI 3 (C# / .NET)**

- **핫패스(파일 열거·검색 인덱싱·미리보기 디코딩·VFS·AI 임베딩)는 Rust 네이티브 코어**로 구현 →
  무 GC, fearless concurrency, 메모리 안전, 최상위 성능. 추후 다른 OS로 이식 용이.
- **UI는 WinUI 3 (XAML, GPU 가속, Fluent Design)** → Microsoft 공식 모던 네이티브 UI.
  Windows Shell API(아이콘/컨텍스트 메뉴/썸네일/속성) 통합이 1급.
- **Rust ↔ .NET 인터롭**: C ABI (`cdylib`) + `NativeLibrary`/P-Invoke, 또는
  필요 시 [`csbindgen`](https://github.com/Cysharp/csbindgen)으로 바인딩 자동 생성.
  대용량 데이터는 복사 대신 공유 버퍼/스팬으로 전달.

> 대안: UI까지 네이티브 극단 성능이 필요하면 C++/WinRT. 단 개발 속도 손해가 커서
> 1차 버전은 C# WinUI로 진행하고, 프로파일링 후 병목 화면만 선택적 C++ 전환을 검토한다.

### 근거 요약

- "성능 최우선"은 대부분 **I/O·인덱싱·디코딩** 병목 → 이 부분만 네이티브(Rust)로 확실히 잡으면
  체감 성능의 90%를 확보. UI 프레임워크 차이는 상대적으로 작다.
- WinUI 3는 **네이티브**이며(웹뷰 아님) 셸 통합·접근성·고DPI를 기본 제공.
- Rust 코어는 단위 테스트·벤치마크·이식성이 좋아 장기 유지보수에 유리.

## 3. 핵심 모듈 (Rust 코어)

| 모듈 | 책임 | 핵심 기술 |
| --- | --- | --- |
| `vfs` | 모든 저장소를 통일된 인터페이스로 추상화 (`Provider` trait) | 로컬/SFTP/S3/OneDrive 어댑터 |
| `indexer` | 파일명·메타·내용 인덱싱, 증분 갱신 | `tantivy`(전문검색), FS watcher 연동 |
| `search` | 질의 → 결과 스트림 (파일명/내용/자연어) | tantivy + (선택) 임베딩 ANN |
| `preview` | 썸네일·프리뷰 디코딩, 캐시 | `image`, PDF, 코드 하이라이트, 캐시 LRU |
| `operations` | 복사/이동/삭제/이름변경, 진행률·취소·실행취소 | 잡 큐, 휴지통, 충돌 정책 |
| `watcher` | 디렉터리 변경 감시 → UI 갱신 이벤트 | `notify` 크레이트 |
| `ai` | 임베딩 생성, 자동 분류/태깅, 자연어 질의 라우팅 | 로컬 모델 or 외부 API 추상화 |
| `interop` | C ABI 표면, 이벤트 스트림, 에러 매핑 | `csbindgen`, 핸들 기반 API |

## 4. UI 셸 (WinUI 3) 핵심 컴포넌트

- **AppShell**: 윈도우, 타이틀바 커스텀, 테마(Fluent/Mica).
- **TabService**: 탭 생성/이동/복원, 각 탭 = 독립 네비게이션 스택.
- **PaneHost**: 단일/듀얼(좌우·상하) 레이아웃, GridSplitter, 활성 패널 포커스.
- **FileListView**: `ItemsRepeater` 기반 **가상화** 리스트/그리드/디테일 뷰.
- **PreviewPane**: 코어 preview 모듈 결과 표시.
- **CommandBar / Hotkeys**: 키보드 우선 워크플로우, 명령 팔레트.
- **StatusBar**: 선택 개수/용량, 백그라운드 잡 진행률.

## 5. 동시성 & 성능 전략

- **비동기 코어**: 모든 I/O는 코어의 스레드풀/잡 큐에서 처리, UI 스레드는 절대 블록하지 않음.
- **가상화 렌더**: 화면에 보이는 항목만 생성. 100k+ 항목도 일정 메모리.
- **스트리밍 열거**: 폴더 진입 시 전체 스캔을 기다리지 않고 도착하는 대로 점진 렌더.
- **다단 캐시**: 메타/썸네일/프리뷰 LRU 캐시 + 디스크 캐시.
- **증분 인덱싱**: watcher 이벤트로 인덱스 부분 갱신, 전체 재스캔 회피.
- **백프레셔**: 잡 큐 우선순위(가시 영역 > 예열 > 백그라운드).

## 5-1. 상주(Always-on) 리소스 관리 · 무간섭 · 오류 격리

> "켜두고 계속 쓰는 앱" — 저메모리, 주기적 정리, 다른 프로그램 무방해, 오류 비전파가 핵심 NFR
> (NFR-M2~M4, NFR-N1, NFR-R3~R5).

### 메모리 절감 & 주기적 정리 (NFR-M2/M3/M4)
- **상한 있는 캐시**: 메타/썸네일/프리뷰/인덱스 캐시는 **하드 캡 + LRU**. 무제한 증가 금지.
- **백그라운드 자니터(Janitor)**: 주기·유휴 트리거로 만료 캐시 정리, 임시 버퍼 반환, 단편화 완화.
- **유휴/최소화 시 작업집합 트림**: 비활성 시 캐시 축소 + `SetProcessWorkingSetSize`/`EmptyWorkingSet`로
  RSS 반납(트림 후 < 80MB 목표). 활성 복귀 시 지연 재적재.
- **메모리 압박 구독**: `QueryMemoryResourceNotification`/저메모리 신호 시 캐시 자발 해제 →
  다른 앱이 메모리를 쓰도록 양보.
- **누수 0 지향**: Rust 코어(소유권)로 코어측 누수 차단, 핸들/COM 참조 수명 추적, soak 테스트로 검증.

### 무간섭(전경 앱 방해 금지) (NFR-N1/R5)
- **저우선순위 백그라운드**: 인덱싱/썸네일 예열/원격 동기화는 **낮은 CPU 우선순위 + Background I/O priority**.
- **스로틀링·백프레셔**: 디스크/네트워크 사용을 제한, 전경 사용자 작업이 항상 우선.
- **비차단 설계**: UI 스레드 블록·전역 락 금지. 모든 장시간 작업은 취소 가능한 비동기 잡.
- **공유 자원 예의**: 파일 핸들은 가능한 공유 모드로 열어 다른 프로그램의 접근을 잠그지 않음.

### 오류 격리(앱/타 프로그램 비중단) (NFR-R3/R4)
- **하위시스템 결함 봉쇄**: 미리보기 디코더·원격 provider 등 위험 작업의 실패가 앱 전체로 전파되지 않게
  결과를 `Result`로 감싸고 패닉 경계(`catch_unwind`)·태스크 단위 복구.
- **위험 디코딩 격리(P2)**: 신뢰 못 할 파일 파싱(썸네일/문서)은 **격리 워커 프로세스**에서 수행 →
  크래시·행이 발생해도 메인 앱과 OS·다른 프로그램에 무영향, 워커 재시작.
- **워치독·우아한 저하**: 응답 없는 작업 타임아웃·취소, 기능 일부 실패 시 나머지는 계속 동작.
- **자동 복구**: provider 재연결, 인덱스 손상 시 부분 재구축, 세션 안전 저장.

## 7. 개발 환경 주의 (macOS)

- 개발 머신은 macOS이나 **WinUI 3 빌드/실행은 Windows 필요**.
  - 권장: Windows 11 + Visual Studio 2022 (Windows App SDK) 환경(물리 PC / VM / CI).
  - Rust 코어는 macOS에서도 `cargo test`/벤치 가능(이식성 설계 이점).
- **CI**: GitHub Actions `windows-latest` 러너에서 코어(cargo) + 앱(msbuild) 빌드, MSIX 패키징.
