## 클래식 설치기 setup.exe — CI 빌드·릴리스 첨부 (PKG-5, `feat/packaging-setup` 브랜치)

> 요구 흐름(사용자): "setup 파일 배포 구성" → "로컬 말고 GitHub Actions로" → "(로컬 Inno 제거) MSIX 서명 방법?" → "GitHub에서 exe 패키징 release"

- **경위**: MSIX(PKG-4)는 서명이 설치 필수 조건 — 서명 조사 결과 **Azure Artifact Signing(구 Trusted Signing)은 현재 개인=미국/캐나다만이라 한국 개인 불가**, 현실 경로=①Microsoft Store 제출(서명 위임, 계정 1회 $19) ②OV 클라우드 서명 구매(연 $100~400) ③자가서명(테스트 전용). → 서명 확정 전 설치형 배포 채널로 **클래식 설치기(Inno Setup)** 채택(PKG-5).
- **구성**: `scripts/nexa-setup.iss`(사용자 단위 기본 `PrivilegesRequired=lowest`+관리자 선택 허용 · 시작 메뉴/바탕화면(선택) · 언인스톨러[사용자 데이터 보존] · `AppId` 고정=업그레이드 식별자 · 라이선스 표시) + `scripts/make-setup.ps1`(포터블과 동일 self-contained 게시, **마커 없음=설치형 표준 AppData 경로** · ISCC 탐색 PATH→Program Files (x86)→사용자 설치).
- **CI**: `package` job에 setup 단계 추가(Inno=windows 러너 기본 포함, 로컬 설치 불요 — 사용자 지시) · 아티팩트 `packages-win-x64`(dist 전체) · 태그 시 릴리스에 **zip+setup.exe 모두 자동 첨부**.
