## B-13u Undo/Redo — 실기 QA 통과·마감 (셸 삭제 가로채기 + 배경 메뉴 항목)

- **실기 QA(사용자) 전부 확인**: 삭제(휴지통→복원) · 복사 · 이동 · 새 폴더 생성 각각 Ctrl+Z/Y 정상. **B-13u S1+S2 검증 완료.**
- **셸 메뉴 "삭제" 가로채기**(`8c6816d`): 셸 메뉴 삭제가 `InvokeCommand`로 셸이 직접 수행 → `DeleteBatchOp` 미기록 → Ctrl+Z가 이전 작업을 되돌리던 버그. `ShellContextMenu`에 **`verbInterceptor` 훅**(선택 후 `GetCommandString(GCS_VERBW)`로 canonical verb 조회) 추가, 호스트가 **"delete" 동사를 `DeletePaths`(휴지통)로 라우팅** — undo 기록·상태바·재로드 통합. 미구현 확장(null verb)은 기존대로 셸 실행. (훅은 후속 cut/copy/paste 앱 클립보드 통합에도 재사용 기반.)
- **빈 영역 메뉴에 실행 취소/다시 실행**(`4cdb708`): "실행 취소: {마지막 작업 설명}"·"다시 실행: …" + Ctrl+Z/Y 표기, 히스토리 없으면 비활성 — 탐색기 배경 메뉴 동일.
