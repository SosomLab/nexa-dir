## 경로 복사(Copy as path) 커스텀 항목 — 셸 동사 제자리 대체 (`feat/copy-as-path` 브랜치)

> ⏱ 2026-07-08 22:05:41 (`3d766de`, 병합 PR#5 `1bbce3f`) · 설계 [docs/38 §7-5](../38-adr-0005-shell-context-menu.md)

- **문제(사용자 보고)**: 교차폴더 다중선택(현재 폴더 2개 + 인라인 확장 자식 폴더 2개 = 4개)에서 셸 "Copy as path"가 **한 폴더의 2개만 복사**. 원인 = 셸 `IContextMenu`가 단일 부모 폴더만 표현 가능 → 호스트가 선택을 `sameDir`로 축소([MainWindow](../app/Nexa.App/MainWindow.xaml.cs) 1271~).
- **해결 = 고유 항목 + 셸 동사 제자리 대체**: `CmItemDef.ReplaceVerb="copyaspath"` 항목은 셸 섹션 대신, 셸 HMENU에서 그 verb(GCS_VERBW) 위치를 찾아 **삭제+삽입**(`VerbReplacement`·`FindMenuPosByVerb`·`DeleteMenu`/`InsertMenuW` MF_BYPOSITION). **같은 자리**에 우리 항목이 뜨고, **축소 이전 전체 선택(`CmCtx.Targets`)** 을 직접 클립보드로 → 교차폴더 4개 전부.
- **Alt 변형**(Q3 실용형): 기본=`"경로"` 줄바꿈(역슬래시) · **Alt(여는 시점 캡처 `CmCtx.Alt`)=POSIX(`/`)·따옴표 없이**. Windows 관용(수정자=여는 시점 판정)과 동일. 실시간 모프는 네이티브 팝업 한계로 별도 트랙.
- **폴백/설정**: 셸이 verb를 안 내면(Shift 없는 Win10 등) 고유 섹션 하단 표시. 설정에서 항목 제거 시 셸 원본 동사 그대로 노출.
- **실기 QA(사용자) 통과**: A-1(교차폴더 4개 전부)·A-2(Alt POSIX)·A-3(제자리 대체·중복 없음) 확인.
- **후속 수정(실기 피드백, 로컬 빌드+실행 검증·CI green)**:
  - 라벨 `"경로 복사"` → **`"Copy as path"`**(제자리 대체라 셸 원 명칭과 시각적 연속, `fix/copy-as-path-label` PR#6 `42435fd`).
  - **화면 표시 순서 정렬**(`fix/copy-as-path-order` PR#7 `1ca8b14`): 폴더/파일 혼합 시 삽입 순서로 나와 화면과 어긋나던 문제 — `ContextTargets`가 `SelectedPaths`(삽입 순서) → 코어 가시 인덱스(`IndexOfPath`) 순으로 반환(`OrderByDisplay`, 비가시=뒤·안정 정렬). Copy as path·Checksum 등 목록 출력이 화면과 일치.
