### (B) 추천 기술 구조 (개발·배포 쉬움)
- **2-tier 실행**: T1 인프로세스 샌드박스(**WASM Component/WIT**, Lua) + T2 아웃오브프로세스 RPC(**Python/Node**, JSON-RPC).
- **선언적 매니페스트 + contribution points**(코드 최소) + activation events(지연 로드).
- **언어별 SDK**(pip `nexa-plugin` / npm `@nexa/plugin` / lua / WIT 바인딩)로 RPC 은닉.
- **Capability 권한 + 리소스 한도 + 서명** → 격리로 상주 안정성(R3/N1/M2).
- **배포**: 서명 번들 `.nexaext` + **Git 기반 레지스트리**(부트스트랩) → 마켓플레이스, 자동 업데이트.
- **DX**: 스캐폴딩 CLI(`nexa plugin create`), 핫리로드, 타입드 API/문서/샘플, 테스트 하니스.
