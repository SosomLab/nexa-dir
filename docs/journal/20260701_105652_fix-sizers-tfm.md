# 작업 기록 — 2026-07-01 10:56:52 (KST)

> 기록 ID: `20260701_105652_fix-sizers-tfm`
> 이전 기록: `20260701_020741_layout-change-record`

## 1. 증상
- Windows에서 `dotnet build app/Nexa.App -c Debug` 실패(4 errors). 레이아웃 골격(다른 PC/macOS 작성, Windows 미검증)의 첫 Windows 빌드.
- `CS0234: 'WinUI' does not exist in 'CommunityToolkit'`(MainWindow.g.i.cs) → XamlCompiler.exe exit 1(MSB3073).
- `CS0108: 'MainWindow.Visible(bool?)' hides 'Window.Visible'`(경고).

## 2. 원인
- **TFM 불일치**: 앱 TFM `net8.0-windows10.0.19041.0` ↔ CommunityToolkit.WinUI.Controls.Sizers **8.1.240916**이 제공하는 Windows TFM은 `18362`/`22621`뿐(**19041 없음**). → XAML 컴파일러가 Sizers 어셈블리 참조를 못 잡아 `CommunityToolkit.WinUI.Controls.GridSplitter` 미해결.
- `Visible(bool?)` 헬퍼가 `Window.Visible` 속성명과 충돌.

## 3. 조치
- **csproj TFM**: `net8.0-windows10.0.19041.0` → **`net8.0-windows10.0.22621.0`**(Sizers 제공 TFM과 정확 매칭). 실행 최소는 `TargetPlatformMinVersion 10.0.17763.0` 유지.
- **MainWindow.xaml.cs**: 헬퍼 `Visible(...)` → `Vis(...)` 로 개명(코드비하인드, XAML 무관).
- 문서 경로 갱신: `docs/18`·`docs/16`의 `...19041.0` → `...22621.0`(journal 과거 기록은 보존).

## 4. 검증
- `dotnet build app/Nexa.App -c Debug` → **Build succeeded, 0 Warning / 0 Error**.
- 출력: `app/Nexa.App/bin/Debug/net8.0-windows10.0.22621.0/`(이전 19041 폴더는 잔여 — 무해).

## 5. 메모
- 레이아웃 골격(F6)이 이제 Windows에서 빌드됨 → 앱 기동/육안 확인 가능(`dotnet run`).
- 교훈: WinUI 앱 TFM은 참조하는 CommunityToolkit 패키지의 제공 TFM과 맞춰야(불일치 시 XAML 컴파일 단계에서 CS0234).
