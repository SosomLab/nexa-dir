# 작업 기록 — 2026-06-30 18:04:58 (KST)

> 이전 기록: `20260630_175714_layout-and-nav-requirements`

## 1. 요청 (사용자)
- 클라우드 파일의 **컨텍스트 메뉴** + **상태(로컬/온라인) 컬럼** 같은 기능을 `ListView` 등 **공통 컴포넌트**로 가능한지, 아니면 **직접 Grid Control**을 만들어야 하는지 검토.
- 특히 핵심인 **트리 인라인 폴더 확장 + 다중 레벨(교차) 파일 선택**에 별도 Grid Control이 필요한지.

## 2. 검토 결과 (결정 — ADR-0002, [21](21-adr-0002-fileview-control.md))
- 플래그십은 **4요건 동시 충족** 필요: 트리(계층) · 다중 컬럼 · 폴더 교차 선택 · 10만 가상화.
- 기성 단일 컴포넌트 비교: ListView(트리✕) · TreeView(컬럼·교차선택·대규모✕) · DataGrid(트리✕) → **4요건 동시 충족 불가**.
- **결정: `ItemsRepeater` 기반 커스텀 트리-그리드(`FileTreeGrid`)**. 기성 TreeView/DataGrid 미채택.

## 3. 목표 / 핵심 설계
- **트리 → 평면화**: 코어가 가시 행(VisibleRow) 평면 스트림 산출 → 컨트롤은 평면 리스트만 가상화(트리 모름). 가상화가 평면 리스트 문제로 환원 → 10만 가능.
- **컬럼**: 행 Grid(공유 컬럼)+헤더. **상태 컬럼=VFS sync 메타**(로컬/온라인/동기중).
- **교차선택**: 코어 `OrderedSet<NodeId>`(부모 무관) 직접 관리.
- **셸 메뉴**: 행 우클릭 `MenuFlyout` + `IContextMenu` 호스팅(클라우드 항목 포함).
- 부분 재사용: 가상화=ItemsRepeater, 리사이즈=CommunityToolkit Sizers, 메뉴=COM 호스팅.

## 4. 영향
- 코어(Rust): VisibleRow 스트림·Selection·expand/collapse API([07](07-flagship-tree-multiselect.md) §3·4), VFS `Entry`에 sync 상태 필드(FR-G).
- 앱(C#): 커스텀 `FileTreeGrid`(현재 ItemsRepeater가 출발점).
