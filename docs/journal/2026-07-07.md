# 2026-07-07 · 작업 기록 (시간 역순)

> 일자 단위 기록. **최신이 위**. 요약 [DEVLOG](../DEVLOG.md)·[MILESTONES](../MILESTONES.md).

## 내장 터미널 UX 일괄 개선 — 캐럿(BUG-007)·포커스·키 매핑·faint 예측(BUG-008 부분)·고정폭 렌더·ECH

한 세션에서 실사용 시나리오(입력→예측→백스페이스)를 따라가며 문제를 연쇄 해결. 각 단위:

### 7) Backspace 키 매핑 — 1글자 삭제로
- **증상**: 백스페이스 1번에 입력 전체가 지워짐.
- **원인**: `\b`(0x08=^H)를 전송 → ConPTY가 **Ctrl+Backspace(단어 삭제, BackwardKillWord)** 로 해석.
- **해결**: `OnKeyDown`에서 `VirtualKey.Back` → **DEL(0x7F)**(1글자), **Ctrl+Backspace → 0x08**(단어 삭제) — 실제 터미널과 동일 매핑. `OnCharacterReceived`의 `\b` 중복 처리 제거.

### 6) ECH(CSI n X)·커서 저장/복원 — 백스페이스 잔상 제거
- **증상**: `dot` 입력 후 백스페이스 → 캐럿은 이동하나 글자·이전 예측이 화면에 잔존.
- **원인**: ConPTY가 라인 일부 지우기에 쓰는 **ECH(`CSI n X`)** 미구현(EL/ED만 지원).
- **해결**: `VtScreen`에 ECH 구현(커서 위치부터 n칸 지움, 커서 유지) + **DECSC/DECRC(ESC 7/8)·CSI s/u** 커서 저장/복원 실제 구현(기존 무시).

### 5) 고정폭 셀 렌더 — 캐럿·열 정렬 (+ 가운데 정렬 회귀 수정)
- **증상**: 캐럿 위치가 글자와 어긋남(특히 `➜` 글리프·한글 이후 누적 드리프트).
- **원인**: 라인을 가로 StackPanel에 run 이어붙이기 → 폴백 폰트 글리프 폭이 고정폭과 달라 열이 누적 밀림. 한글(전각)을 버퍼에서 1칸으로 취급해 셸(2칸 계산)과 불일치.
- **해결**: 라인을 **Canvas 절대 배치**(run을 열×CharW에 고정)로 교체 — ASCII run은 묶고, 비ASCII는 글자별 자기 열 고정. `VtScreen.Put`에 **전각 2칸 전진**(wcwidth 근사 `IsWide` + 연속 셀 `'\0'`). 배경 Border도 셀폭 고정.
- **회귀**: 명시 Width+기본 Stretch 정렬이 **가운데 배치**로 동작 → `HorizontalAlignment.Left` 고정으로 수정.

### 4) faint(SGR 2) 렌더 — PSReadLine 예측을 연한 회색으로 (BUG-008 부분 해결)
- **진단**: 내장 터미널은 `pwsh`(프로파일 로드)를 구동 — `$PROFILE`의 oh-my-posh(robbyrussell)+PSReadLine 확인. "과거 history 보임"의 정체 = **PSReadLine 인라인 예측**(faint로 출력)인데 faint 미반영이라 실제 입력처럼 진하게 보였던 것.
- **해결**: `TermCell.Faint` 추가(셀 보관·run 분리 포함) → 렌더에서 **Opacity 0.45** — VS Code처럼 연한 회색 미리보기. 미사용 `_priv` 정리(경고 0).

### 3) 클릭 포커스 안정화
- **증상**: 터미널 클릭 시 포커스가 잡혔다 풀림(마우스 release 시점).
- **원인**: 동기 `Focus()`가 WinUI 기본 포인터/포커스 처리에 덮임 + 내부 ScrollViewer가 이벤트를 Handled 처리하면 XAML 속성 핸들러 미호출 + **release 기본 처리가 포커스 재탈취**.
- **해결**: `PointerPressed`/`PointerReleased`를 **`AddHandler(handledEventsToo:true)`** 로 등록하고 `FocusSoon()`(dispatcher enqueue — 기본 처리 이후 재포커스).

### 2) Tab 자동완성 — 포커스 이동 차단
- **증상**: 터미널에서 Tab이 자동완성 대신 다른 컴포넌트로 포커스 이동.
- **해결**: `OnKeyDown`에 Tab 추가(`Handled=true`로 WinUI 포커스 내비 차단) — Tab=`\t`, **Shift+Tab=`ESC[Z`**(역방향 완성). `OnCharacterReceived`의 `\t` 중복 제거.

### 1) 캐럿(커서) 표시 — BUG-007 해결
- `VtScreen`에 커서 좌표 노출(`CursorCol`/`CursorRow`/`ScrollbackCount`).
- `TerminalView`에 **캐럿 오버레이**(CaretLayer Canvas) — **포커스 시 반투명 accent 블록+깜빡임(530ms)**, **비포커스 시 외곽선(중공)** = 포커스 여부 시각화. 출력 직후 즉시 표시. `GotFocus`/`LostFocus`로 전환.

검증: 각 단위마다 로컬 빌드(`dotnet build app/Nexa.App`) green + 앱 실행 수동 확인(이 PC는 WinUI 로컬 실행 가능). 최종 **경고 0·에러 0**.
