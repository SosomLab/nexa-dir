# 02 · 로드맵

단계는 "동작하는 제품"을 빠르게 만들고 점진 확장하는 순서로 구성한다.
**Milestone 1 = 탭 + 듀얼 패널** (사용자 지정 1순위).

---

## M0 · 기반 구축 (Foundation)

> 목표: 빈 화면이라도 코어↔UI가 연결되어 폴더를 연다.

- [ ] 리포 구조 확정 (`core/` Rust 워크스페이스, `app/` WinUI 3 솔루션)
- [ ] Windows 빌드 환경 + GitHub Actions CI (`windows-latest`)
- [ ] Rust 코어 ↔ C# 인터롭 PoC (`csbindgen`로 함수 1개 왕복)
- [ ] `nexa-vfs` 로컬 provider: 디렉터리 **스트리밍 열거** API
- [ ] WinUI 셸: 단일 창 + `ItemsRepeater` 가상화 리스트로 로컬 폴더 렌더
- [ ] 기본 네비게이션(더블클릭 진입, 뒤로/앞으로/위로, 주소줄)

**완료 기준**: 임의의 로컬 폴더를 열어 항목을 가상화 렌더하고 탐색할 수 있다.

---

## M1 · 탭 + 듀얼 패널 ★ (1순위)

> 목표: 핵심 차별화 워크플로우 완성.

### 탭 (패널별, Double Commander / Path Finder 식)
- [ ] `TabService`: 탭 추가/닫기/순서변경/복제, Ctrl+T/W/Tab 단축키
- [ ] **각 패널 상단 자체 탭 스트립**, 싱글 라인 ↔ 멀티 라인 전환
- [ ] 탭별 독립 네비게이션 히스토리(뒤로/앞으로)
- [ ] 세션 복원(앱 재시작 시 열린 탭/경로/펼침·선택 복구)
- [ ] 중간클릭으로 폴더를 새 탭에서 열기

### 컨텍스트 메뉴 (셸 통합 + 고유 메뉴)
- [ ] Windows 셸 컨텍스트 메뉴 호스팅(`IContextMenu`/`IExplorerCommand`)
- [ ] Nexa 고유 항목 병합(새 탭/패널 열기·선택 바구니·경로 복사 등)
- [ ] 단일 액션 레지스트리(메뉴·단축키·명령 팔레트 공유)

### 퀵 런처 바 (통합 터미널 + 외부 프로그램)
- [ ] 타이틀바 하단 아이콘 런처 스트립(활성 경로 컨텍스트)
- [ ] 통합 터미널(cmd/pwsh/WT) 현재 폴더에서 실행
- [ ] 외부 프로그램 등록/실행(경로·인자 토큰 `%path%`/`%selection%`·아이콘)
- [ ] 추가/삭제/순서변경, 파일 드래그앤드롭으로 열기
- [ ] (P2) 임베디드 터미널 패널, 런처 프로파일

### 듀얼 패널
- [ ] `PaneHost`: 좌우/상하 분할, `GridSplitter`로 비율 조정
- [ ] 활성 패널 포커스 표시, Tab 키로 패널 전환
- [ ] **패널 간 복사/이동**(F5/F6, 드래그앤드롭) — 두 경로 동시 표시의 핵심 가치
- [ ] 단일 ↔ 듀얼 토글, 레이아웃 상태 저장

### 인라인 폴더 확장 + 폴더 교차 다중 선택 ★ 플래그십 (P0)
> 상세: [07-flagship-tree-multiselect.md](07-flagship-tree-multiselect.md)
- [ ] 코어: 트리 → **가시 노드 평면 스트림** 모델 + 지연 펼침(스트리밍 열거)
- [ ] UI: `ItemsRepeater` 가상화 트리(삼각형 펼침/접힘, 들여쓰기)
- [ ] **폴더 교차 다중 선택**(Ctrl/Shift, OrderedSet<NodeId>), 혼합 하이라이트
- [ ] 혼합 선택 → 복사/이동/삭제/드래그(작업 엔진 혼합 입력) 연동
- [ ] 펼침/선택 상태 정렬·외부변경·세션 복원에 일관 유지

### 파일 작업 (기본)
- [ ] `nexa-ops`: 복사/이동/삭제(휴지통)/이름변경 + 진행률·취소
- [ ] 충돌 처리(덮어쓰기/건너뛰기/이름변경), 실행취소(Undo)
- [ ] FS watcher 연동으로 외부 변경 자동 반영
- [ ] 정렬/그룹/뷰 모드(목록/디테일/타일/그리드)

**완료 기준**: 탭 여러 개 + 듀얼 패널 + **인라인 폴더 확장**으로,
**서로 다른 폴더의 파일을 동시에 다중 선택**해 드래그/단축키로 이동·복사하고,
작업 진행률과 실행취소가 동작한다.

## M2 · 빠른 미리보기 (Quick Preview)

- [ ] `nexa-preview` 디코더: 이미지(JPEG/PNG/WebP/HEIC), 텍스트/코드(하이라이트), PDF 1페이지
- [ ] Windows 썸네일 공급자 연동(셸 썸네일) + 자체 캐시(LRU + 디스크)
- [ ] `PreviewPane` + 스페이스바 퀵룩 스타일 오버레이
- [ ] 메타데이터 패널(EXIF/문서 속성/해시)

**완료 기준**: 선택 즉시(<100ms) 일반 포맷 미리보기와 메타 표시.

## M3 · 빠른 검색 & 인덱싱

- [ ] `nexa-index`(tantivy): 파일명·경로·메타 증분 인덱싱, watcher 연동
- [ ] 인스턴트 파일명 검색(Everything 유사, <50ms)
- [ ] 내용(전문) 검색, 필터(타입/크기/날짜/태그) 문법
- [ ] 검색 결과를 가상 폴더처럼 탐색/작업

---

## M4 · 클라우드 / 네트워크 / 원격 통합 (VFS) — 차근차근 확장

> 공통 `Provider` trait을 먼저 안정화하고, provider를 **단계적으로** 추가한다.

- [ ] **단계 1**: SMB/CIFS(UNC)·매핑 네트워크 드라이브, SFTP
- [ ] **단계 2**: S3 호환 오브젝트 스토리지, WebDAV/FTP(S)
- [ ] **단계 3**: 클라우드 SaaS — OneDrive → Google Drive → Dropbox/Nextcloud
- [ ] 자격증명 안전 저장(Windows Credential Manager/DPAPI), 사이드바 마운트
- [ ] 비동기 스트리밍·부분 다운로드·캐시, 오프라인 표시
- [ ] 원격 ↔ 로컬, 원격 ↔ 원격 전송(듀얼 패널·교차 선택과 결합)

## M5 · AI 검색 / 정리

- [ ] `nexa-ai`: 임베딩 생성(로컬 모델 or 외부 API 추상화)
- [ ] 자연어 검색("지난주 받은 송장 PDF")
- [ ] 자동 분류·태깅·중복/유사 파일 탐지
