# 15 · 개발 방법론 — 작은 단위 · 점진 프로토타이핑 · 단위별 커밋

> 요구: 구현은 **작은 기능 단위로 하나씩 순차 진행**, **단위별 커밋**으로 분리 분석 가능,
> 각 단위 **초안 개발 후 확장하는 프로토타이핑(incremental) 방식**으로 개발/테스트.

---

## 1. 핵심 원칙

1. **수직 슬라이스(Vertical slice)** — 한 단위는 *관찰·테스트 가능한* 얇은 끝단(코어→인터롭→UI) 슬라이스.
   레이어 전체(가로)가 아니라 "동작하는 최소 기능"(세로).
2. **작게·순차로** — 한 번에 한 단위, 의존 순서대로.
3. **초안 먼저, 그다음 확장(프로토타이핑)** — 최소 동작 초안 → 검증 → 점진 확장. 각 확장도 작은 단위.
4. **항상 빌드 가능(green)** — main은 늘 빌드·테스트 통과. 각 단위가 코어/앱을 깨지 않음.
5. **단위 = 커밋** — 커밋 하나가 하나의 일관·리뷰가능·분석가능 단위(작은 diff).
6. **테스트 동반** — 각 단위에 테스트(코어 단위/통합) 포함, 통과 후 다음 단위.

## 2. 단위(Unit)의 정의

- **무엇을**: 사용자/개발자가 *관찰 가능한 최소 변화* 1개 (예: "폴더 행을 인라인으로 펼친다").
- **포함**: 구현 + 테스트 + (필요 시) 문서/journal 한 줄.
- **크기 기준**: 리뷰 10분 내, diff가 한 화면 수준이면 이상적. 커지면 쪼갠다.
- **완료(DoD)**: 빌드/테스트 통과 · 의도한 동작 확인 · 커밋.

## 3. 단위 1개 작업 루프

```
1) 정의   : 가장 작은 관찰가능 슬라이스 + 수용 기준 1~2개
2) 초안   : 최소 구현(가장자리는 stub 가능)
3) 검증   : cargo test / dotnet test / 실행 확인
4) 커밋   : 단위 1개 = 커밋 1개  (초안)
5) 확장   : 케이스·에러·성능·UX 추가 → 각 확장도 별도 커밋
6) 다음 단위로
```

## 4. 커밋 규약 (분리 분석 가능하도록)

- **Conventional Commits**: `type(scope): 요약`
  - type: `feat` `fix` `refactor` `test` `perf` `docs` `build` `ci` `chore`
  - scope: 모듈 — `core/vfs` `core/index` `core/pty` `interop` `app/tree` `app/tabs` `app/pathbar` …
- **한 커밋 = 한 단위**. 초안과 확장은 **나눠서** 커밋(예: `(초안)` → 기능 추가).
- 예:
  - `feat(core/vfs): 디렉터리 스트리밍 열거 초안`
  - `test(core/vfs): 대용량/빈/권한오류 케이스`
  - `feat(app/tree): 인라인 펼침 렌더 초안`
  - `feat(app/tree): 펼침 상태 정렬 후에도 유지`
- 본문: 무엇/왜 + 검증 방법 1줄. 푸터에 Co-Authored-By 유지.

## 5. 브랜치/리뷰

- 기능 묶음마다 **단명 브랜치**(`feat/m1-pathbar`) → 작은 커밋 누적 → **PR로 main 병합**.
  (혹은 신뢰 시 main 직접 작은 커밋. 단 main은 항상 green.)
- PR/커밋은 작게 유지 → **단위별 분석·되돌리기 용이**.

## 6. 프로토타이핑(incremental) 운영

- **초안(prototype)**: 동작 우선, 가장자리는 단순/stub. "보이게" 만들고 빠르게 검증.
- **확장(iterate)**: 초안 위에 케이스·성능(NFR)·엣지·UX를 한 단위씩 덧붙임.
- 큰 미지수는 **스파이크**(버릴 수 있는 실험)로 먼저 검증 후 정식 단위화.

## 7. M0 → 초기 M1 단위 백로그 (순차)

> 순서대로 한 단위씩. 각 항목 ≈ 1커밋(필요 시 초안/확장으로 분할).

**M0 (기반)**
1. `chore(repo)`: 리포 스캐폴딩(core 워크스페이스 + app 스켈레톤 + .gitignore)
2. `ci`: GitHub Actions(windows-latest 빌드/test, macos 코어 test) 골격
3. `feat(interop)`: Rust↔C# 왕복 PoC(함수 1개) + 테스트
4. `feat(core/vfs)`: 로컬 디렉터리 **스트리밍 열거** 초안 + 테스트
5. `feat(app/list)`: `ItemsRepeater` 가상화로 폴더 1개 렌더(초안)
6. `feat(app/nav)`: 진입/뒤로/앞으로/위로

**M1 (1순위 묶음, 각 기능 초안→확장)**
7. `feat(app/pathbar)`: 계층 경로 바 초안(세그먼트 표시/클릭 이동) → 확장(형제 드롭다운·경로 편집·드롭)
8. `feat(app/tree)`: 인라인 폴더 펼침 초안(지연 로드) → 확장(상태 유지·대규모 가상화)
9. `feat(core/select)`+`feat(app/select)`: **폴더 교차 다중 선택** 초안 → 확장(Shift 범위·중복 흡수)
10. `feat(core/ops)`: 혼합 선택 복사/이동/삭제 초안 → 확장(진행률·충돌·Undo)
11. `feat(app/tabs)`: 패널별 탭 초안 → 확장(멀티라인·세션 복원)
12. `feat(app/dual)`: 듀얼 패널 초안 → 확장(F5/F6·드래그)
13. `feat(app/ctxmenu)`: 셸 컨텍스트 메뉴 호스팅 초안 → 확장(고유 항목 병합)
14. `feat(app/launcher)`: 퀵 런처 바 초안 → 확장(등록·드롭)
