# 05 · 전체 요구사항 (목표 수집)

> 개발 착수 전 **전체 목표를 수집**하기 위한 단일 출처(Single Source of Truth).
> 기능(FR)·비기능(NFR)·제약·가정·리스크를 망라한다. 변경 시 journal에 기록.
> 표기: `[M#]` 목표 마일스톤, `[P0/P1/P2]` 우선순위(P0=MVP 필수).

---

## 1. 범위 (Scope)

- **In:** Windows 데스크톱 파일 관리자 앱 — 탐색/작업/미리보기/검색/원격/AI.
- **Out(초기):** 파일 편집기 내장, 시스템 셸 완전 대체, macOS/Linux 정식 배포.
- **타깃 OS:** Windows 11 (Windows 10 22H2 best-effort).

## 2. 사용자 페르소나 & 핵심 시나리오

| 페르소나 | 핵심 작업 | 대표 시나리오 |
| --- | --- | --- |
| 개발자/파워유저 | 다계층 폴더, 원격 서버, 키보드 중심 | 듀얼 패널로 로컬↔SFTP 대량 이동, Git 폴더 탐색 |
| 크리에이터 | 대량 미디어 정리·미리보기 | 수천 장 사진 빠른 프리뷰·태깅·중복 제거 |
| 일반 사용자 | 빠르고 직관적 탐색 | 다운로드 폴더 자동 분류, 빠른 검색 |

## 3. 기능 요구사항 (Functional)

### FR-A 탐색 / 네비게이션 `[M0/M1] P0`
- FR-A1 로컬 폴더 **스트리밍 열거 + 가상화 렌더**(100k+ 항목).
- FR-A2 **★ 계층 경로 바(브레드크럼) — 우선 구현 대상.** 폴더 단위로 구분된 전체 경로를 세그먼트로 **계층 분리 표시**
  (Macintosh HD › Users › … 식). 각 세그먼트 **클릭 이동**, 세그먼트 **드롭다운(▾)으로 형제 폴더 전환**,
  세그먼트에 **드래그앤드롭**, 활성 패널/탭과 동기.
  - FR-A2b **우클릭 → 텍스트 주소 모드**: 전체 경로를 편집 가능한 텍스트로 전환 → **복사/붙여넣기/직접 입력 후 Enter 이동**.
- FR-A3 뒤로/앞으로/위로 네비게이션. **뒤로/앞으로 버튼 우클릭 → 히스토리 목록 → 2단계 이상 점프**(브라우저식). 히스토리는 **탭별**(FR-B2).
- FR-A4 **뷰 모드(교체형)**: **아이콘**(크기 조절)/**목록(디테일)**/**계층(컬럼·Miller)**/**갤러리**, 정렬·그룹·필터. 같은 데이터·선택 공유([22](22-adr-0003-view-and-panel-modules.md)).
- FR-A5 사이드바: 즐겨찾기/핀/드라이브/최근/마운트.
- FR-A6 **메뉴 바 + 도구 모음(크롬)**: 드롭다운 메뉴(파일/선택/명령/보기/도구/도움말) + 아이콘 단축 **도구 모음**(네비/뷰/작업). **아이콘 16×16 기본, 32×32 옵션**. 도구 모음·메뉴 바 표시/숨김. → 레이아웃 [20](20-ui-layout.md).
- FR-A7 **하단 도킹 패널 모듈**: 상세정보(단일=메타 / 다중=파일 목록·합계)·미리보기·Hex·**터미널(Cmd/Pwsh)** 등 모듈을 **배치/위치/크기/표시 선택** + 세션 저장. 교체·확장형 추상화([22](22-adr-0003-view-and-panel-modules.md)).
- FR-A8 **컬럼 시스템**: **기본 정의열**(이름/**확장자 분리 옵션**/크기/종류/날짜/속성/**클라우드 상태**/경로) + **사용자 정의열(수식·문자열 조합, 안전 샌드박스)** + 플러그인열. 컬럼 추가/제거/순서/너비/표시·정렬, 레이아웃 저장·복원, **듀얼 패널 동기화(독립/실시간/수동 적용 선택)**. → [23](23-column-system.md).

### FR-B 탭 (패널별) `[M1] P0`
> Double Commander / Path Finder 식 — **각 패널 상단에 자체 탭 스트립**.
- FR-B1 탭 생성/닫기/복원/재배치, 단축키(Ctrl+T/W/Shift+T/Tab).
- FR-B2 탭별 독립 히스토리, 중간클릭으로 새 탭.
- FR-B3 세션 저장·복원(탭/경로/스크롤/펼침·선택 상태).
- FR-B4 **싱글 라인 ↔ 멀티 라인 탭 스트립** 전환(탭이 많을 때 줄바꿈으로 모두 표시).
- FR-B5 탭을 패널/창 간 드래그 이동(P1).

### FR-C 듀얼 패널 `[M1] P0`
> Double Commander / Path Finder 식 — **좌/우 분리 패널이 각각 다른 디렉터리**를 탐색.
- FR-C1 좌우/상하 분할, 스플리터 비율, 단일↔듀얼 토글.
- FR-C2 활성 패널 포커스, Tab 전환, **패널마다 독립적인 탭 스트립(FR-B)**.
- FR-C3 패널 간 복사/이동(F5/F6, 드래그앤드롭).
- FR-C4 (P1) 동기 스크롤·폴더 비교(diff).

### FR-X 인라인 폴더 확장 + 폴더 교차 다중 선택 `[M1] P0` ★ 플래그십
> **제1 차별화 포인트.** 상세 설계: [07-flagship-tree-multiselect.md](07-flagship-tree-multiselect.md).
- FR-X1 macOS Finder 목록 보기처럼 현재 폴더에서 **하위 폴더를 인라인으로 펼침/접힘**(진입과 별개).
- FR-X2 **서로 다른 부모 폴더의 파일을 동시에 다중 선택**(Ctrl/Shift 클릭, 교차폴더).
- FR-X3 혼합 선택을 한 번에 복사/이동/삭제/드래그/압축(작업 엔진 혼합 입력).
- FR-X4 펼침/선택 상태를 정렬·외부변경·세션 복원에도 일관 유지.
- FR-X5 대규모 가상화 트리(10만 가시 노드)에서 NFR-P1/P2 충족.

### FR-D 파일 작업 `[M1] P0`
- FR-D1 복사/이동/삭제(휴지통/영구)/이름변경/새 폴더/바로가기.
- FR-D2 백그라운드 잡: 진행률·속도·ETA·일시정지·취소.
- FR-D3 충돌 정책(덮어쓰기/건너뛰기/둘 다/최신만).
- FR-D4 **실행취소(Undo)**, 휴지통 경유 기본, 대량 삭제 확인.
- FR-D5 (P1) **일괄 이름변경 — Path Finder식 배치 리네임**(규칙 스택: 찾기바꾸기/추가/제거/대소문자/일련번호/날짜·EXIF/템플릿 + **실시간 미리보기** + 충돌 검출 + Undo, **교차폴더 선택 결합**). 상세 [25](25-bulk-rename.md).

### FR-E 미리보기 `[M2] P1`
- FR-E1 이미지/문서/코드/미디어 프리뷰, 메타데이터 패널.
- FR-E2 스페이스바 퀵룩 오버레이 + 사이드 PreviewPane.
- FR-E3 썸네일(셸 + 자체) 캐시.

### FR-F 검색 `[M3] P1` — 상세 [24](24-search-everything.md)
> **Everything식 인스턴트 파일명 검색**(NTFS MFT 일괄 열거 + USN 실시간 갱신, <50ms). 구현 대상 등록.
- FR-F1 **인스턴트 파일명 검색**(전 드라이브, 인메모리 인덱스, 실시간 갱신). 비NTFS/원격은 walkdir+watch 폴백.
- FR-F2 필터 문법(type/size/date/tag/path) + 부분/와일드카드/정규식 매칭.
- FR-F3 내용(전문, tantivy) 검색, 결과를 **가상 폴더**로 탐색/작업(듀얼 패널·교차 선택과 결합).

### FR-G 클라우드 / 네트워크 / 원격 드라이브 `[M4] P2`
> **목표로 설정하고 단계적(차근차근) 구현.** VFS provider 모델로 점진 확장.
- FR-G1 **네트워크 드라이브**: SMB/CIFS(UNC `\\server\share`), 매핑된 드라이브, WebDAV.
- FR-G2 **원격**: SFTP/FTP(S), S3 호환 오브젝트 스토리지.
- FR-G3 **클라우드**: OneDrive → Google Drive → Dropbox/Nextcloud (단계적).
- FR-G4 자격증명 안전 저장(Credential Manager/DPAPI), 사이드바에 마운트로 노출.
- FR-G5 비동기 스트리밍·부분 다운로드·캐시·오프라인 표시.
- FR-G6 전송(로컬↔원격, 원격↔원격) 재개·체크섬 검증, 듀얼 패널·교차 선택과 결합.

> **단계 제안:** ① SMB/네트워크 드라이브 + SFTP(가장 수요 큼) → ② S3 → ③ 클라우드 SaaS.
> provider 1개를 깊게 완성한 뒤 다음으로 확장(공통 `Provider` trait 안정화 우선).

### FR-H AI `[M5] P2`
- FR-H1 자연어 검색, 임베딩 기반 의미 매칭.
- FR-H2 자동 분류·태깅·중복/유사 탐지.
- FR-H3 정리 규칙 추천 + 일괄 적용 미리보기.

### FR-J 컨텍스트 메뉴 (셸 통합 + 고유 메뉴) `[M1] P0`
> **기본 Windows 컨텍스트 메뉴를 그대로 사용** + **Nexa 고유 메뉴 항목 추가 표시**.
- FR-J1 Windows **셸 컨텍스트 메뉴** 호스팅(`IContextMenu`/`IExplorerCommand`) —
  서드파티 셸 확장 항목(열기 앱·압축·백신 등) 포함.
- FR-J2 Nexa **고유 항목 추가**(새 탭/패널에서 열기, 선택 바구니에 추가, AI 작업 등)
  를 셸 메뉴와 **병합**(앞/뒤 또는 서브메뉴).
- FR-J3 Windows 11 모던 메뉴/클래식 메뉴 모두 대응, 다중·교차폴더 선택에서도 동작.
- FR-J4 단축키·명령 팔레트와 동일 액션 공유(단일 액션 레지스트리).

### FR-K 퀵 런처 바 (Quick Launch Toolbar) `[M1] P1`
> **타이틀바 바로 아래 최상단에 아이콘 런처 영역** — 통합 도구/외부 프로그램을 빠르게 실행.
- FR-K1 타이틀바 하단에 **이미지 아이콘 버튼 스트립**(가로 배치, 활성 패널 경로 컨텍스트).
- FR-K2 **통합 터미널 실행**(M1): cmd / PowerShell / pwsh / Windows Terminal —
  **현재 활성 폴더 경로에서** 외부 실행.
- FR-K2b **★ 하단 임베디드 터미널 패널** — Path Finder처럼 창 **하단에 도킹된 터미널 패널**(cmd/pwsh).
  **설계는 지금 우선 반영, 구현은 후속(P2)**. 활성 패널 경로 동기(cd), 다중 터미널 탭, 표시/숨김 토글.
  상세 설계: [03 §3-K2b](03-features.md), Windows **ConPTY** 호스팅.
- FR-K3 **외부 프로그램 실행**: 텍스트 에디터 등 사용자가 등록(실행파일 경로·인자·아이콘) →
  현재 폴더 또는 **선택 파일을 인자로** 실행.
- FR-K4 **커스터마이즈**: 항목 추가/삭제/순서변경, 아이콘 지정, 설치 프로그램 자동 감지(선택).
- FR-K5 인터랙션: 클릭 실행, **파일을 아이콘에 드래그앤드롭**해 그 프로그램으로 열기,
  단축키 바인딩, 컨텍스트 메뉴(FR-J)·명령 팔레트와 동일 액션 공유.
- FR-K6 (P2) 작업 프로파일/그룹(개발·미디어 등 용도별 런처 세트 전환).

### FR-L 플러그인 / 확장 (스크립트) `[M6] P2`
> Python/Node/Lua/WASM 스크립트로 기능 확장. 상세: [09-plugin-architecture.md](09-plugin-architecture.md).
- FR-L1 **확장 포인트**: 명령·컨텍스트 메뉴·커스텀 컬럼·미리보기·VFS 공급자·검색·자동화 규칙·AI 액션 등.
- FR-L2 **실행 모델 2-tier**: 인프로세스 샌드박스(WASM/Lua) + 아웃오브프로세스 RPC(Python/Node).
- FR-L3 **선언적 매니페스트 + contribution points**(코드 최소), activation events(지연 로드).
- FR-L4 **언어별 SDK**(pip/npm/lua/WIT)로 개발 친화, 스캐폴딩 CLI·핫리로드.
- FR-L5 **Capability 권한 + 리소스 한도 + 서명** — 격리로 상주 안정성(R3/N1/M2) 보장.
- FR-L6 **배포**: 서명 번들 + Git 기반 레지스트리 → 마켓플레이스, 자동 업데이트.

### FR-M 내장 압축 (zip) `[향후 지원]` P2
> **기본 zip 압축을 내장**(외부 도구 의존 없이). **향후 지원 기능** — 지금 우선순위 아님.
- FR-M1 선택 항목(**교차폴더 다중 선택 포함**)을 zip으로 **압축 생성**.
- FR-M2 zip **압축 해제**(전체/선택), 대상 폴더 지정·충돌 정책.
- FR-M3 (후속) zip을 **폴더처럼 인플레이스 탐색**(VFS provider로 마운트).
- FR-M4 진행률·취소·Undo(작업 엔진 연동). 구현: Rust `zip` 크레이트(Deflate, 퍼미시브 라이선스).
- 비고: 1차는 **zip만 내장**, 7z/rar 등은 플러그인([09](09-plugin-architecture.md))로 확장.

## 4. 비기능 요구사항 (Non-Functional) — 측정 가능

| ID | 항목 | 목표치 |
| --- | --- | --- |
| NFR-P1 | 폴더 첫 렌더(100k 항목) | < 150ms |
| NFR-P2 | 스크롤 프레임 | 60fps 유지 |
| NFR-P3 | 미리보기(일반 포맷) | < 100ms |
| NFR-P4 | 파일명 검색 응답 | < 50ms |
| NFR-P5 | 콜드 스타트 | < 1s |
| NFR-M1 | 대용량 폴더 메모리 | 가상화로 항목 수와 무관하게 상한 |
| **NFR-M2** | **상주 유휴 메모리** (항상 켜두는 앱) | 유휴 시 작업집합 **< 150MB**, 트림 후 **< 80MB** (목표, 측정·조정) |
| **NFR-M3** | **주기적 정리** | 백그라운드 자니터가 캐시 LRU 정리·작업집합 트림(유휴/최소화 시), 누수 0 |
| **NFR-M4** | **메모리 압박 대응** | 시스템 메모리 부족 시 캐시 자발적 해제로 다른 앱 방해 안 함 |
| **NFR-N1** | **무간섭(성능)** | 백그라운드 작업은 **저우선순위 CPU + Background I/O**, 스로틀링으로 전경 앱 방해 금지 |
| NFR-R1 | 안정성 | 파괴적 작업 무손실(트랜잭션·Undo·휴지통) |
| NFR-R2 | 크래시 | UI 크래시가 진행 중 파일 작업을 손상시키지 않음 |
| **NFR-R3** | **오류 격리** | 한 하위시스템(미리보기 디코더·원격 provider 등) 오류가 **앱 전체/다른 프로그램을 중단·차단하지 않음** |
| **NFR-R4** | **장시간 안정성** | 수일 연속 구동에도 메모리·핸들 누수 없이 안정(soak 테스트 통과) |
| **NFR-R5** | **비차단** | 전역 락·UI 스레드 블록 없음, 어떤 작업도 OS/다른 앱을 잠그지 않음 |
| NFR-S1 | 보안 | 자격증명 DPAPI/Credential Manager, 최소 권한 |
| NFR-A1 | 접근성 | 키보드 완전 조작, 스크린리더, 고DPI |
| NFR-Q1 | 품질 | 코어 단위 테스트 + 핵심 경로 통합 테스트, CI 게이트 |

## 5. 제약 (Constraints)

- C1 개발 머신은 macOS — WinUI 빌드/실행은 Windows(빌드 머신/CI) 필요.
- C2 네이티브 + 최대 성능 요구(사용자 명시).
- C3 Windows Shell 통합(아이콘/컨텍스트 메뉴/썸네일/IFileOperation) 활용.
- C4 **VSCode로 개발 가능**(풀 Visual Studio IDE 강제 없이) — CLI 빌드(`cargo`/`dotnet`/MSBuild) 우선,
  필요한 SDK·VS Build Tools는 winget 설치 허용. → 판단: [06 §7-2(a)](06-adr-0001-stack.md).
- C5 **GitHub Actions로 패키징·배포 가능**(`windows-latest`에서 빌드+MSIX+서명+Releases 자동화).
  → 판단: [06 §7-2(b)](06-adr-0001-stack.md).

## 6. 가정 (Assumptions)

- A1 1차 사용자는 Windows 11, x64/ARM64.
- A2 초기엔 단일 사용자 데스크톱(서버/멀티유저 아님).
- A3 AI는 옵트인, 로컬 우선 — 외부 API는 동의 후.
