# 20 · UI 레이아웃 (영역 · 크기 · 숨김)

> 화면 영역 골격을 **기능 구현 전에** 확정한다. 각 영역의 목적·크기 변경 여부·숨김 가능 여부를 정의하고,
> 현재는 **placeholder 패널**(라벨/테두리)로 화면에 식별 가능하게 둔다. 디자인 방향 [10](10-decision-record.md) §3, 컴포넌트 [01](01-architecture.md) §4.

## 2. 영역 정의

| 영역 | 목적 | 위치 | 크기 변경 | 숨김 | 단계 |
| --- | --- | --- | :---: | :---: | --- |
| **메뉴 바** | 드롭다운 메뉴(파일/선택/명령/보기/도구/도움말) | 최상단 | 고정 높이 | (옵션) | M1 |
| **도구 모음**(아이콘) | 단축 버튼(네비 `←→↑⟳` · 뷰 전환 · 작업). **아이콘 16×16 기본, 32×32 옵션** | 메뉴 아래 | 고정 높이 | ○ | M1 |
| **퀵 런처 바**(전역) | 외부 터미널/에디터 등 빠른 실행 | 도구모음 아래 | 고정 높이 | ○ | M1 |
| **탭 바**(패널별) | 탭 표시·전환·생성, 패널별 싱글/멀티라인 | 패널 상단 | 고정 높이 | ✕ | M1 |
| **네비게이션 바**(패널별) | **뒤로/앞으로/위로** + 경로 바(브레드크럼)·경로 편집 | **탭 바로 하단** | 고정 높이 | ✕ | M1 |
| **좌 패널(주)** | 파일 리스트/인라인 트리. 항상 1개 이상 | 중앙 좌 | 가변(좌우 splitter) | ✕ | M0~M1 |
| **우 패널(듀얼)** | 보조 파일 영역. 단일↔듀얼 토글 | 중앙 우 | 가변(좌우 splitter) | ○ | M1 |
| **미리보기 패널** | 선택 파일 미리보기 | 우측/패널 내 | 가변 | ○ | M2 |
| **하단 도킹 패널** | **상세/미리보기/Hex/터미널(ConPTY)** 모듈 — 배치·위치·크기·표시 선택([22](22-adr-0003-view-and-panel-modules.md)) | 하단/우/플로팅 | 가변(splitter) | ○ | M1~ |
| **상태바** | 선택 수·용량, 잡 진행률, (현재) 숨김 토글 | 최하단 | 고정 높이 | ✕ | M0~ |
| (타이틀바) | 커스텀 타이틀·창 컨트롤·Mica | 창 상단 | 고정 | ✕ | 후속 |

- **크기 변경**: 가변 영역은 `GridSplitter`(CommunityToolkit Sizers) 드래그로 조절. 숨길 때 해당 splitter도 함께 숨겨 빈 공간을 남기지 않는다. 좌/우 패널 `MinWidth`로 과축소 방지.
- **경로 바는 패널별**(전역 아님): 듀얼 패널이 서로 다른 경로를 가지므로 각 패널의 네비게이션 바에 둔다.

## 3. 네비게이션 & 히스토리 (탭별) — ★ 구현 대상

> 브라우저식. 요구 [05](05-requirements.md) FR-N*, 백로그 [15](15-dev-methodology.md) §7. 영역 = 패널 네비게이션 바.

- **탭별 독립 히스토리**: 각 탭 = 독립 back/forward 스택. 탭 전환 시 그 탭의 위치·히스토리 복원.
- **[←] 뒤로 / [→] 앞으로**: 1단계 이동. 새 위치로 진입하면 forward 스택 초기화(브라우저와 동일).
- **우클릭 히스토리 점프**: ← / → 버튼을 **우클릭하면 히스토리 목록(MenuFlyout)** 표시 → 항목 선택으로 **2단계 이상 점프**.
- **[↑] 위로**: 부모 폴더로 이동(히스토리에 누적).
- **경로 바(브레드크럼) & 주소 편집**: 폴더 단위로 구분된 전체 경로를 **세그먼트**로 표시 → 세그먼트 클릭 이동, 세그먼트 `▾` = 형제 폴더 드롭다운. **우클릭 → 텍스트 주소 모드**: 전체 경로를 편집 가능한 텍스트로 전환 → **복사/붙여넣기/직접 입력 후 Enter 이동**.
- **상태 저장**: 히스토리·현재 위치를 탭/세션에 저장·복원.
- 버튼 활성/비활성: 스택이 비면 ← / → 비활성.

## 4. 현재 구현 상태 (골격)

- 모든 영역을 **placeholder 패널**(배경 틴트 + 라벨)로 배치 — 화면에서 영역 즉시 식별.
- 좌 패널: 탭 바(placeholder) → 네비바(`[←][→][↑]` 버튼 + 경로 텍스트, placeholder) → **실제 디렉터리 목록**(F4/F5).
- **크기 조절**: 좌↔우 패널, 메인↔터미널 splitter 동작.
- **숨김**: 상태바 토글로 런처 바·우 패널·터미널 패널 표시/숨김(splitter 동반).
- 네비게이션 히스토리 **로직은 미구현**(버튼은 placeholder) → 탭/네비게이션 인프라 단위에서 구현.
- 구현: [app/Nexa.App/MainWindow.xaml](../app/Nexa.App/MainWindow.xaml) · 검증 [19](19-implemented-features.md) F6.

## 5. 뷰 모드 & 도킹 패널 (→ [22](22-adr-0003-view-and-panel-modules.md) ADR-0003)

- **뷰 모드(교체형 `IFileView`)**: **아이콘**(크기 조절) / **목록(디테일)** / **계층(컬럼·Miller)** / **갤러리**(하단 스트립+상단 큰 미리보기). 같은 데이터·선택을 공유, 도구 모음에서 전환.
- **하단 도킹 패널(`IToolPanel` + `PanelHost`)**: 상세정보(단일=메타 / 다중=파일 목록·합계) · 미리보기 · Hex · 터미널(Cmd/Pwsh). **배치/위치/크기/표시 선택 + 세션 저장**, 활성 선택 구독.
- 둘 다 **추상화 계층(인터페이스)** 으로 교체·확장(플러그인 [09](09-plugin-architecture.md) 확장점).

## 6. 후속(영역별 채움 순서)
좌 패널 네비게이션(폴더 진입·↑) → 경로 바(브레드크럼) → **탭별 히스토리(←/→·우클릭 점프)** → 탭 모델 → 우 패널 듀얼 → 인라인 트리([07](07-flagship-tree-multiselect.md)) → 미리보기(M2) → 터미널(ConPTY).
