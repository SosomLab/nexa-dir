# 07 · 플래그십 기능 — 인라인 폴더 확장 + 폴더 교차 다중 선택

> **이 프로젝트의 제1 차별화 포인트.** macOS Finder의 "목록 보기(List view)"처럼,
> 더블클릭으로 들어간 폴더 안에서 **하위 폴더를 그 자리에서 펼쳐(disclosure triangle)** 파일을 보고,
> **서로 다른 폴더에 있는 파일들을 동시에 다중 선택**해 한 번에 작업할 수 있다.
> Windows 기본 탐색기는 이 두 가지를 모두 못 한다 → 핵심 우위.

관련: [05-requirements.md](05-requirements.md) FR-C/FR-D, [06-adr-0001-stack.md](06-adr-0001-stack.md).

---

## 1. 사용자 관점 동작 (Behavior)

### 1-1. 인라인 폴더 확장 (Outline / Tree-in-List)
- 디테일/목록 보기에서 각 폴더 행 좌측에 **펼침 삼각형(▶/▼)**.
- 삼각형 클릭 또는 `→`(펼침)/`←`(접힘)으로 **현재 위치를 떠나지 않고** 하위 내용을 들여쓰기로 표시.
- 더블클릭 = 그 폴더로 **진입(네비게이트)**, 삼각형/단축키 = **인라인 펼침**. (두 동작 공존)
- 여러 폴더를 동시에 펼친 상태 유지, 깊이 제한 없음(성능 허용 범위).
- 펼침 상태는 탭/세션에 저장·복원.

### 1-2. 폴더 교차 다중 선택 (Cross-folder Multi-selection) ★ 최우선
- 펼쳐진 트리에서 **서로 다른 부모 폴더의 항목들**을 함께 선택.
- `Ctrl+클릭`(비연속 토글), `Shift+클릭`(가시 순서상 범위), `Ctrl+A`(현재 뷰 가시 노드 전체).
- 선택은 **경로 집합**이며 깊이/부모가 달라도 무방.
- 선택한 혼합 집합에 대해 **복사/이동/삭제/드래그/압축** 등 일괄 작업.
  - 예: A폴더의 사진 3장 + B폴더의 PDF 2개를 함께 선택 → C로 이동.

### 1-3. 보기 모드와의 관계
- 인라인 확장은 **목록/디테일 보기** 전용(그리드/타일은 평면 유지, 펼침 비활성 또는 진입만).
- 정렬은 **부모 단위로 적용**(각 펼친 폴더 내부에서 정렬), 트리 구조는 보존.

## 2. Windows 탐색기 대비

| 동작 | Windows 탐색기 | Nexa Dir |
| --- | --- | --- |
| 파일 목록 영역에서 폴더 인라인 펼침 | ✕ (좌측 내비 트리만) | ✅ |
| 서로 다른 폴더의 파일 동시 선택 | ✕ | ✅ |
| 혼합 선택 일괄 작업 | ✕ | ✅ |

## 3. 데이터 모델 (Core, Rust)

핵심 아이디어: **트리를 "가시 노드의 평면 스트림"으로 투영**하여 가상화 렌더 + 빠른 선택을 동시에 달성.

```
NodeId        = 안정적 고유 식별자 (경로 정규화 + provider id; 경로변경 추적용 inode/파일ID 병용)
Node {
  id: NodeId,
  parent: NodeId?,
  name, kind(file|dir|symlink), depth,
  meta(size, mtime, ...),
  state: { expanded: bool, loaded: bool, childCount? }
}
VisibleRow = { id, depth, expanded, hasChildren, meta }   // UI로 흘려보내는 단위
```

- **펼침 = 지연 로딩**: `expand(id)` → VFS로 해당 폴더 **스트리밍 열거** → 자식 노드 추가 →
  변경된 가시 구간만 UI에 diff 이벤트로 전송(insert/remove/range).
- **가시 스트림 생성**: 루트의 펼친 노드를 DFS 순회하되 **가상화 윈도우(보이는 구간)만** 구체화.
  대규모 트리는 부모별 자식 개수 캐시로 행 인덱스↔노드 매핑을 O(log n)에 계산.
- **정렬/필터**: 부모 노드별 정렬 비교자 적용, 펼친 자식은 그 비교자 순서로 삽입.

## 4. 선택 모델 (Selection)

```
Selection = OrderedSet<NodeId>      // 삽입 순서 보존(작업 순서·드래그 미리보기용)
anchor: NodeId?                     // Shift 범위의 기준
```

- **불변식**: 선택은 *임의의* NodeId 집합 — 같은 부모일 필요 없음(=교차 선택의 핵심).
- **Shift 범위**: 트리 평면화 순서(visible order)의 anchor~current 사이 가시 노드.
- **경로 변동 대응**: 외부 변경(이동/삭제)으로 노드가 사라지면 선택에서 자동 제거 + 이벤트.
- **폴더 선택 시맨틱**: 폴더 자체를 선택하면 작업은 폴더(재귀)를 대상으로 함. 펼친 자식도 따로
  선택돼 있으면 **중복 제거**(부모가 선택되면 그 하위는 작업 대상에서 흡수).

## 5. 작업 엔진 연동 (Operations)

- 작업 입력은 항상 **소스 경로 리스트(혼합 허용)** + 대상 1곳.
- 서로 다른 볼륨/Provider가 섞이면 작업을 **그룹핑**(로컬↔로컬 rename-fast, 교차볼륨 copy+delete,
  로컬↔원격 전송)하여 단일 잡의 하위 태스크로 스케줄.
- 충돌/진행률/취소/Undo는 혼합 집합 전체를 하나의 트랜잭션 묶음으로 처리.

## 6. 드래그 앤 드롭

- 드래그 페이로드 = 선택된 **NodeId/경로 집합**(다중 부모 가능).
- 듀얼 패널/다른 탭/외부 앱으로 드롭 시 표준 셸 데이터(`CF_HDROP`)로 변환.
- 드롭 대상이 펼친 트리의 특정 하위 폴더면 그 폴더가 대상.

## 7. UI 렌더 (WinUI 3)

- **자체 트리**: 기본 `TreeView` 대신, 코어의 `VisibleRow` 스트림을 `ItemsRepeater`로 가상화.
  - 행 = 들여쓰기(depth) + 삼각형 + 아이콘 + 컬럼(이름/수정일/크기/종류).
  - 펼침/접힘은 코어에 명령 → 돌아온 diff로 행 삽입/삭제 애니메이션.
- **선택 시각화**: 다중·비연속·교차폴더 하이라이트(스크린샷의 파란 하이라이트 다중 행과 동일 경험).
- **성능 목표**: NFR-P1/P2 — 10만 가시 노드에서 첫 렌더 <150ms, 스크롤 60fps.

## 8. 키보드 / 인터랙션 규약

| 키 | 동작 |
| --- | --- |
| `→` / `←` | 펼침 / 접힘 (자식 없으면 부모로 이동) |
| `↑` / `↓` | 가시 행 이동 |
| 더블클릭 / `Enter` | 폴더 진입(네비게이트) |
| `Ctrl+클릭` | 비연속 토글 선택(교차폴더) |
| `Shift+클릭` / `Shift+↑↓` | 가시 범위 선택 |
| `Ctrl+A` | 현재 뷰 가시 노드 전체 선택 |
| `Space` | 빠른 미리보기(M2) |

## 9. 엣지 케이스

- 매우 큰 폴더 펼침 → 스트리밍 + "더 불러오는 중" 표시, 부분 렌더.
- 순환 심볼릭 링크 → 방문 집합으로 사이클 차단.
- 권한 없는 폴더 → 펼침 시 인라인 에러 행.
- 펼친 중 외부 변경(watcher) → 해당 부모 구간만 부분 갱신, 선택/펼침 상태 보존.
- 정렬 변경 → 펼침/선택 유지한 채 재정렬.

## 10. 수용 기준 (Acceptance)

- AC1 목록 보기에서 임의 폴더를 인라인으로 펼치고 접을 수 있다(진입과 별개).
- AC2 서로 다른 부모 폴더의 항목을 동시에 선택할 수 있다.
- AC3 혼합 선택을 한 번에 복사/이동/삭제하고 진행률·Undo가 동작한다.
- AC4 혼합 선택을 드래그하여 다른 패널/폴더로 드롭할 수 있다.
- AC5 10만 가시 노드에서 NFR-P1/P2(첫 렌더<150ms, 60fps)를 만족한다.
- AC6 펼침/선택 상태가 정렬·외부변경·세션 복원에도 일관되게 유지된다.

## 11. 마일스톤 반영

- **M1(탭/듀얼 패널)과 동급 P0**로 승격. 트리 모델·교차 선택은 M1 범위에 포함.
- M0 스파이크에서 *가상화 트리 + 교차 선택* 성능을 우선 검증(ADR-0001 §7).
